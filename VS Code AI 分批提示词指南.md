## 核心原则

1. **先骨架后血肉** — 先搭项目框架和公共组件，再逐模块生成页面
2. **先简单后复杂** — 先做纯展示页、列表页，再做流程页、交互页
3. **每批一个焦点** — 每次 Prompt 只聚焦 1-2 个页面，避免上下文溢出
4. **带上规范前缀** — 每次 Prompt 开头都引用规范文档，保证风格一致
5. **累积上下文** — 后续批次引用前面已生成的类型定义和公共组件
6. **Mock 数据驱动** — 每个页面同步生成 mock 数据文件，Demo 可独立运行
7. **跨模块跳转** — PRD 中 StatCard、图表、待办列表等大量组件需点击跳转至其他模块页面（共 30+ 处），统一使用 React Router `navigate()`，Demo 阶段确保路由连通

## 客户 UAT 验收场景

<aside>
🧪

客户验收时自行操作的业务场景脚本。每个场景可独立验收打勾。

</aside>

### UAT-1：用户注册与登录闭环

1. 访问官网首页 → 浏览核心能力和数据亮点
2. 点击「立即注册」→ 填写注册表单 → 获取验证码 → 提交
3. 注册成功自动跳转工作台
4. 退出登录 → 使用账号密码重新登录 → 进入工作台
5. 验证：切换手机验证码登录方式也能登录 ☐

### UAT-2：智能体接入全流程

1. 以科室管理员登录 → 进入接入中心
2. 点击「新增注册」→ 填写基本信息 + 技术信息 + 上传备案材料 → 提交
3. 查看注册记录列表 → 状态变为「对接中」→「对接成功」
4. 进入台账列表 → 确认新接入智能体已出现，状态为「已接入」
5. 验证：对接失败场景 → 修正重提可走通 ☐

### UAT-3：准入评测与上线

1. 以平台管理员登录 → 进入评测任务管理 → 发起评测
2. 选择智能体 → 选择四维评测 → 提交
3. 查看评测报告 → 确认雷达图 + 各维度明细 + 结论
4. 点击「批准准入」→ 台账中该智能体状态变为「已上线」
5. 验证：评测指标库可新增/停用指标 ☐

### UAT-4：诊疗场景使用闭环

1. 以普通用户登录 → 工作台场景入口
2. 选择就诊类型「门诊」→ 选择阶段「接诊中」→ 出现场景卡片
3. 点击场景卡片 → 打开对话窗口 → 发送消息 → 收到智能体回复
4. 点击收藏 → 确认出现在「我的收藏」区域 → 再次一键打开
5. 验证：编排流程场景显示进度条和分步引导 ☐

### UAT-5：多智能体编排协同

1. 以平台管理员登录 → 进入编排中心 → 新建场景
2. 新建流程 → 拖拽智能体节点 + 人工审核节点 → 连线 → 保存发布
3. 场景配置绑定编排流程 → 工作台场景入口可触发
4. 验证：预置的「门诊全流程协同」流程（8 步）可正常展示 ☐

### UAT-6：运行监控与告警处置

1. 切换至「异常告警态」→ 数据大屏出现告警红色高亮
2. 点击告警卡片 → 跳转监控中心告警列表
3. 查看告警详情 → 点击「处理」→ 填写处理方案 → 状态变为「已处理」
4. 验证：监控首页四维 Tab 数据正常、性能排行可下钻 ☐

### UAT-7：安全治理与风险处置

1. 进入安全治理总览 → 六维雷达图正常展示
2. 点击某维度 → 跳转风险处置列表 → 查看风险详情
3. 填写处置方案 → 状态变为「已解决」
4. 验证：安全评分变化反映在总览卡片中 ☐

### UAT-8：智能体注销下线闭环

1. 以科室管理员发起注销申请 → 选择智能体、注销原因
2. 系统自动生成影响评估 → 显示关联流程/调用量/风险等级
3. 以平台管理员审批 →「批准下线」
4. 台账中该智能体状态变为「已注销」
5. 验证：审计日志中记录完整操作链路 ☐

### UAT-9：数据大屏全屏投屏

1. 进入数据大屏 → 点击全屏按钮
2. 确认：深色主题、16:9 布局、图表自动轮播（10s）、隐藏导航
3. ESC 退出全屏 → 恢复正常模式
4. 验证：自动刷新（5 分钟）生效 ☐

## 进阶技巧

### 1. 在项目根目录放规范文件

将《AI 代码生成规范参考文档》保存为项目中的规则文件，AI 工具会自动读取：

```
Cursor  →  .cursorrules
Claude  →  CLAUDE.md
Copilot →  .github/copilot-instructions.md
Windsurf → .windsurfrules
```

这样每次 Prompt **无需重复粘贴规范**，只需说「遵循项目规范」即可。

### 2. 上下文管理策略

```
方法 A（推荐）：@文件引用
  在 Prompt 中用 @src/types/agent.ts @src/components/StatusTag/index.tsx
  让 AI 读取已有代码，确保复用而非重复造轮子

方法 B：摘要回顾
  每开始新阶段时，用一段话总结已完成的内容：
  "已完成：项目初始化、公共组件(StatusTag/StatCard/...)、官网门户6个页面、
   工作台+数据大屏。接下来进入阶段4：接入中心。"
```

### 3. 处理 AI 上下文窗口不够的情况

```
拆分策略：
- 如果一个页面的需求描述超过 2000 字，拆成两次：
  第1次：页面框架 + 静态展示部分
  第2次：交互逻辑 + 状态管理 + Mock 数据

- 如果需要粘贴大量字段表格，可以先让 AI 生成类型定义，
  再在下一轮让它基于类型生成页面组件
```

### 4. 质量检查 Prompt

每完成一个阶段后，运行一次检查：
