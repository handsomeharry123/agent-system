# 医小管-医院资源审核-需求说明

## 一、场景定位

| **项** | **说明** |
| --- | --- |
| 场景名称 | 资源审核 |
| 操作完成方向（预设） | 1.管理员进入即默认开始资源使用权限审核（无需先发起，由医小管自动打招呼、主动呈报待审清单）；
2.沿固定任务轨道 N0→N5 逐条完成 X 条资源使用权限申请审核（X 由 list_pending 返回）；
3.完成标志为每条待审申请均产生明确结论（审核通过/退回修改）并生效、授予或退回、同步用户端并通知申请人；
4.收尾后仅开放动作：① 继续审核下一条 ② 暂停并保存进度（未决项保持待审核） ③ 查看本批次汇总 |
| 进入方式 | 1.意图路由：命中「审核资源申请／资源权限审批／有哪些待审资源申请」等口令，且权限校验通过后进入；
2.快捷入口：对话页常驻快捷指令【资源使用权限审核】（仅管理员可见）；
3.从权限审批页等相关入口进入。无论何种方式，进入即由医小管自动打招呼、主动发起 |
| 完成标志 | 1.每条待审申请均产生明确结论（审核通过/退回修改）并落库生效；
2.按结论完成授予该智能体对接/调用所申请资源的权限，或退回申请人；
3.将审核意见同步至用户端并通知申请人；
4.或管理员暂停，未决项保持待审核状态 |
| 产出物 | 1.每条申请的审核记录：审核结论、具体说明、审核人、审核时间；
2.状态流转：审核通过→授予该智能体对接/调用所申请资源（业务系统/模型）的权限；退回修改→退回申请人；
3.对用户端的同步：审核通过显示为「具体说明」、退回修改显示为「退回原因说明」；
4.对申请人的站内通知；
5.本批次汇总（审核通过 X／退回修改 Y／未处理 Z） |
| 预计步数 / 时长 | 1.单条约 5 步（N1→N5）；
2.单条 2–3 分钟；
3.批次总时长随待审数量线性增长 |
|  |  |

## 二、任务轨道（状态机节点表）

> 每节点独立一行；`动作`只写要点，通用规则不写；`出口判据`是节点被视为已完成的系统条件；`工具`只列本节点会触发的工具调用。
> 

| **节点** | **名称** | **动作（要点）** | **完成条件** |
| --- | --- | --- | --- |
| N0 | 进场与待审呈报 | 1.管理员进入且权限校验通过后自动打招呼，固定开场：「你好，我是医小管。目前资源申请待审核 X 项」（X 由 list_pending 返回）；
2.在下方自动呈现待审清单：智能体编号、智能体名称、申请资源名称（跨业务系统/模型两类） | 1.待审清单已呈现；
2.待审数量已播报 |
| N1 | 选定与信息呈报 | 1.据管理员指令选定一条（如「看第 1 个」「那个内分泌的」）；
2.呈报该条权限申请信息（只读）：智能体编号、名称、所属科室、诊疗环节、功能描述、申请资源名称；
3.附基础核对提示（资源是否已注册可用、是否重复授权、功能与所申请资源相关性、写入/调用范围是否合理），仅供参考不代替裁决 | 1.选定唯一一条待审申请；
2.该条申请信息呈报完成 |
| N2 | 审核结论（收指令） | 1.请管理员给出审核结论：审核通过 / 退回修改（单选、必填）；
2.管理员用自然语言下达，医小管解析并回述确认；
3.据结论联动具体说明填写要求（退回修改必填、审核通过选填） | 1.管理员已给出审核结论；
2.结论落定为「审核通过」或「退回修改」之一 |
| N3 | 具体说明 | 1.收集具体说明，≤500 字并实时显示已输入字数；
2.选「退回修改」时必填（退回修改原因），选「审核通过」时选填；
3.医小管解析并回述确认，管理员说「提交/确认」即进入落库 | 1.具体说明字数合规（≤500 字）；
2.满足所选结论的必填/选填要求；
3.管理员完成确认 |
| N4 | 生效·授权/退回·同步通知 | 1.落库审核记录（审核结论、具体说明、审核人=当前管理员、审核时间）；
2.按结论流转：审核通过→授予该智能体对接/调用所申请资源的权限；退回修改→退回申请人；
3.同步至用户端：审核通过显示为「具体说明」、退回修改显示为「退回原因说明」；
4.通知申请人并回执处理结果；
5.自动回到 N1 处理下一条 | 1.审核记录落库；
2.授权或退回执行完成；
3.用户端同步与站内通知完成 |
| N5 | 批次收尾 | 1.清空待审或管理员暂停时触发；
2.播报本批：审核通过 X 条、退回修改 Y 条、未处理 Z 条；
3.收尾跟进期间记录的待办事项；
4.固定收尾语并释放场景锁 | 1.本批次汇总播报完成；
2.待办事项已答复；
3.场景锁释放 |

轨道全局规则：

1. 结论由管理员做出，医小管不代裁决、不在未给结论时落库；
2. 退回修改必须填写具体说明方可提交；
3. 同一申请不重复出结论；
4. 询问跑偏→固定话术「我们先来完成资源使用权限审核，稍后再为您解决此问题」并复述当前审核事项，可解决项记入待办、收尾统一跟进；
5. 管理员暂停→保存进度、释放场景锁，未决项保持待审核。

## 三、字段清单

> 只列本场景相关字段；类型 / 校验优先复用《平台通用规范 v1》第九章基线；差异在「规则」列写明。
> 

| **字段** | **来源** | **类型** | **规则（差异化）** |
| --- | --- | --- | --- |
| 智能体编号 | 只读引用（来源于申请） | 短文本 | 1.取自智能体注册表单自动生成的编号；
2.格式为「科室编号-准入顺序号」（如 XNK-0001）；
3.本场景只读呈现，不可修改 |
| 智能体名称 | 只读引用（来源于申请） | 短文本 | 1.取自智能体注册表单；
2.超出 10 字省略号显示、悬浮展示完整名称；
3.点击名称跳转「智能体详情页」；
4.本场景只读呈现，不可修改 |
| 所属科室 | 只读引用（来源于申请） | 枚举（院内字典） | 1.取自智能体注册表单，格式为「科室代码+科室名称」；
2.本场景只读呈现，不可修改 |
| 诊疗环节 | 只读引用（来源于申请） | 单选枚举 | 1.取自智能体注册表单选择项：导诊分诊／预问诊／预约挂号／辅助检查／辅助诊断／辅助治疗／住院／手术／其他（含填空）；
2.本场景只读呈现，不可修改 |
| 功能描述 | 只读引用（来源于申请） | 长文本 | 1.取自智能体注册表单的「功能描述」字段；
2.超出 20 字省略号显示、悬浮展示完整内容；
3.本场景只读呈现，不可修改 |
| 申请资源名称 | 只读引用（来源于申请） | 结构化文本（可多值） | 1.取自申请人选择的资源，覆盖业务系统数据资源与模型资源两类，可多选；
2.超出 15 字省略号显示、悬浮展示完整名称；
3.本场景只读呈现，不可修改；
4.授权范围即以本字段为准 |
| 审核结论 | 对话获取（管理员填） | 单选枚举 | 1.管理员对本次权限申请的判定结果；
2.取值单选：审核通过 / 退回修改（必填）；
3.联动「具体说明」的提示文案与填写要求；
4.必须由管理员做出，医小管不代裁决、不默认 |
| 具体说明 | 对话获取（管理员填） | 长文本 | 1.管理员针对审核结论填写的详细意见或退回原因；
2.多行文本，≤500 字并实时显示已输入字数；
3.选「退回修改」时必填、选「审核通过」时选填；
4.提交后同步用户端：退回修改显示为「退回原因说明」、审核通过显示为「具体说明」 |
| 审核人 | 系统生成 | 短文本 | 1.自动记录为当前登录的审核管理员；
2.对话中不涉及录入 |
| 审核时间 | 系统生成 | 日期时间 | 1.系统自动记录审核结论提交的时间；
2.格式为 YYYY-MM-DD HH:MM；
3.对话中不涉及录入 |
| 授权/退回结果 | 动作回填 | 结构化文本 | 1.审核通过→授予该智能体对「申请资源名称」中各资源的使用权限（业务系统对接 / 模型调用）；
2.退回修改→退回申请人待重新提交；
3.由 submit_review 工具执行并回填结果 |
| 用户端同步与通知 | 动作回填 | 状态 | 1.同步至用户端并向申请人发送站内通知；
2.同步字段命名：审核通过→「具体说明」；退回修改→「退回原因说明」；
3.由 submit_review 工具在落库后自动触发 |

特殊情况说明：

1.权限申请信息（智能体编号、名称、所属科室、诊疗环节、功能描述、申请资源名称）均为只读引用，医小管不修改申请内容；管理员若要求修改申请信息，一律拒绝并说明只能由申请人重新提交。

2.管理员若要求医小管代为裁决、或退回修改不填写原因直接退回，一律拒绝：审核结论必须由管理员做出，退回修改必须填写具体说明方可提交。

3.同一申请不得重复出结论；若管理员对已审结申请再次下达指令，医小管提示已完成并可跳转对应审核记录查看。
