# 医小管-医院资源注册-需求说明

## 一、场景定位

| **项目** | **说明** |
| --- | --- |
| 场景名称 | 资源注册 |
| 平台智能体 | 医小管（统一对话页的平台智能体，用户进入场景即弹出资源列表并自动打招呼） |
| 使用角色 | 信息科／资源管理员 |
| 资源两类 | 1. 业务系统数据资源（1.2.1）：院内业务系统（HIS/EMR/PACS 等），含资源、负责人、联系方式、对接方式及技术子字段；
2. 模型资源（1.2.2）：大模型（模型名称、版本、部署方式、API 地址、API key）；
3. 一次会话可登记一个或多个、可两类混合 |
| 操作完成方向（预设） | 1. 用户进入场景即弹出「业务系统数据资源列表」，由医小管自动打招呼；
2. 沿固定任务轨道 N0→N6 完成材料采集、智能识别与类型判定、字段回填与补全、自动访问测试与提交；
3. 完成标志为访问测试全部通过并提交入库后弹出「资源注册成功」；
4. 收尾后仅开放动作：① 继续注册下一条资源 ② 暂存草稿（保留 7 天）③ 资源入库供智能体接入时对接／调用引用 |
| 进入方式 | 1. 意图路由：用户说"注册资源／登记一个系统／接入个模型"等命中进入；
2. 快捷入口：对话页常驻快捷指令【资源注册】；
3. 从台账"更多→医院资源管理中心"进入；
4. 无论何种方式，进入即由医小管弹出两类资源列表、自动打招呼、主动发起 |
| 完成标志 | 1. 资源类型与字段识别／补全完整；
2. 业务系统按对接方式、模型按 API 完成访问测试并通过；
3. 用户指令确认提交后落库并弹出"资源注册成功"，资源入库；
4. 或用户暂存为草稿（保留 7 天） |
| 产出物 | 1. 一条或多条医院资源注册记录（业务系统数据资源／模型资源）；
2. 每条记录含各自字段与访问测试结果；
3. 入库后供智能体接入时对接／调用引用 |
| 预计步数 / 时长 | 约 7 步（N0–N6）、5–10 分钟（视资源数量而定） |

## 二、任务轨道

> **场景类型**：材料驱动型任务——入场后以"提供材料→补全字段→确认提交"逐步推进。
**主流程**：自动开场·弹列表·关键字匹配 → 多模态识别与自动回填 → 缺失字段对话补全 → 自动访问测试 → 汇总确认 → 提交入库并弹出“资源注册成功”。
> 

### 1. 开启对话入口

入口方式：

**方式一：首页对话框直接发送消息（文字/语音），以意图命中自然进入资源注册；**

用户说"注册资源／登记一个系统／接入个模型"等命中口令，直接进入场景，由医小管自动打招呼并邀请提供材料。

**方式二：首页点击业务标签「资源注册」快捷开启对话；**

如果是通过方式二点击业务标签「资源注册」发起对话，由医小管主动弹出「业务系统数据资源列表」与「模型资源列表」（明确告知平台“已支持注册的系统／模型清单”范围），并发送固定开场语：

「你好！我是医小管。把资源信息文件发给我（支持 PDF、DOC、DOCX、XLSX、csv、jpg、jpeg、png、链接等任意文件格式），或文字、语音描述，我来帮你进行医院资源注册～」

同时在列表下方附输入框与提示语：“没找到？直接输入系统关键字（如缩写、别名、业务描述），我来帮你匹配”，支持文字或语音输入关键字。

**两种方式均在左侧「历史会话」新建并进入对应的对话窗口，后续多轮对话在同一窗口内持续进行。无论何种进入方式，进入即由医小管自动打招呼、主动邀请，无需用户先开口。**

### 2. 意图识别

医小管对用户每轮输入先做意图判定，归入以下类别之一，并分流至对应处理：

**意图一：提供材料 / 补全字段（对应 3 执行对应意图 · 意图一）**

- 判定依据：用户上传任意格式材料（单文件≤30M，支持多文件）、或发送文字／语音描述资源信息，或对追问项作答；若用户直接输入系统关键字（缩写／别名／业务描述）同属提供材料；
- 后续处理：进入材料与描述采集、列表外关键字匹配（如适用）、智能识别与资源类型判定、字段回填与缺失字段对话补全流程。

**意图二：确认提交 / 修改字段（对应 3 执行对应意图 · 意图二）**

- 判定依据：字段与资源信息齐备后，用户以点击、语音、文字任一方式发出「提交」意图；或在任一节点用户（点／说／打字）“修改××”；
- 后续处理：修改类先定位目标字段／节点处理后回到中断处；确认类进入自动访问测试 → 汇总确认 → 提交并弹出“资源注册成功”流程；若对接／技术字段或 API 信息被修改，则自动重跑访问测试。

### 3. 执行对应意图

根据 2 意图识别的结果，分流至以下两条执行链路；分支内步骤按 MECE 原则互斥、依序推进。

**3.1 意图一：获取注册信息**

> 通过「列表展示与列表外关键字匹配 → 多模态材料识别与自动回填 → 缺失字段对话补全」三步，完成资源识别与全部必填字段采集。
> 

**A. 列表展示与列表外关键字匹配**

1. 医小管弹出「业务系统数据资源列表」与「模型资源列表」，明确告知平台“已支持注册的系统／模型清单”范围，支持用户在列表内直接挑选；
2. 列表外引导：用户在列表下方输入框以文字或语音输入关键字，调用 match_resource 在院内资源字典（缩写／全称／别名／业务描述）中做模糊匹配，返回 Top-N 候选（默认 3–5 个），按匹配度排序；
3. 确认询问：以候选卡形式反问“您是要注册【XXX 系统（全称）】吗？”，用户可点击、说或打字确认；命中一项→资源槽位落定；多义或均不符→追问一次澄清关键信息（分类、别名、所在科室）后再次匹配。

**B. 多模态材料识别与自动回填**

1. 接收用户上传的任意格式文件（PDF、DOC、DOCX、XLSX、csv、jpg、jpeg、png、链接等，单文件≤30M，支持多文件）、文字或语音，并对每个输入做基础校验（大小、可解析性），不可解析时提示改用其它格式或改为文字描述；
2. 对材料／描述执行多模态识别（OCR／文档解析／ASR／链接抓取）；
3. 判定资源类型（业务系统数据资源／模型资源，可多条、可混合），回填对应字段草稿；
4. 播报每条资源的已填字段与仍缺失项。

**C. 缺失字段对话补全（分支）**

按资源类型逐项对话追问（每轮一问），用户以手动／语音／文字任一方式回答：

1. 业务系统数据资源：资源（从资源列表匹配、多选）、资源负责人（必填，已有用户选或手输）、联系方式（手机或座机）、对接方式（HL7/FHIR/DICOM/数据库直连/MQ）及分支子字段；
2. 模型资源：模型名称、模型版本、部署方式、API 地址、API key（密文）；
3. 密钥／凭据类密文保存，不明文回显；
4. 循环直至各资源必填字段填写完整并校验通过。

**3.2 意图二：确认提交与提交入库**

> 依次进入自动访问测试 → 汇总确认 → 提交入库三步；期间若用户对任一字段“修改××”，定位目标节点处理后回到中断处；技术／API 信息被修改时自动重跑访问测试。
> 

**A. 自动访问测试**

全部资源信息就绪后自动发起访问测试，按资源类型／对接方式分支执行：

1. HL7：连 IP:端口（MLLP／HTTP）；
2. FHIR：请求 URL 校验可达与鉴权；
3. DICOM：对 IP:端口做 C-ECHO；
4. 数据库直连：按类型／IP／端口测试连接；
5. MQ：连 Broker 并校验认证；
6. 模型资源：用 API 地址＋API key 调用连通性测试；
7. 回报成功／失败＋原因，失败定位字段引导修正后自动重测。

**B. 汇总确认**

1. 以对话消息逐项呈现：每条资源的类型、关键字段（密钥密文）、对接／部署配置、访问测试结果，供用户核对；
2. 每项可【修改】；用户点击、语音、文字发出“修改××”→跳转对应节点处理后返回本节点重新汇总；
3. 确认提交同样支持点击、语音、文字，三种输入方式等效。

**C. 提交入库并弹出“资源注册成功”**

1. 提交并落库资源注册记录；
2. 弹出「资源注册成功」回执（各资源名称／类型、对接或部署方式、测试状态）；
3. 资源进入平台资源库，供智能体接入时对接／调用引用。

### 4. 对话延续及收尾

1. 每一步的确认／选择／修改／补填均支持手动操作、语音、文字三种等效方式；
2. 单步完成后医小管保持接收状态，等待用户继续操作（补填、修改、确认提交等）；
3. 提交入库并弹出“资源注册成功”后，本场景自动结束，释放场景锁。

### 5. 特殊情况说明

**特殊情况1：跑偏拉回**

情况定义：任一节点用户询问内容跑偏（含请求其他平台操作），偏离当前医院资源注册。

处理方式：

1. 发送固定话术（不可改写）：“我们先来完成医院资源注册，稍后再为您解决此问题”，随后复述当前问题；
2. 不切换场景、不对无关话题展开解答；
3. 平台内可解决项记入待办，收尾后统一跟进。

**特殊情况2：字段／节点修改**

1. 任一节点用户说“修改××”→定位对应节点处理后回中断处；
2. 若涉及对接／技术字段或 API 信息变更，则自动重跑访问测试。

**特殊情况3：访问测试失败**

1. 首次失败时定位到对应技术字段引导修正后自动重测；
2. 若用户知悉失败后选择修正，重测通过方视为完成；
3. 若用户选择不修正，则可暂存草稿后结束场景（同特殊情况5）。

**特殊情况4：列表外关键字匹配失败**

连续 2 次仍无匹配→固定提示（不可改写）：“目前平台仅支持列表内资源注册，暂不支持自定义资源；如确有需要请联系管理员登记扩展”，并回引导用户在列表内挑选或改换关键字重试。

**特殊情况5：用户暂存／放弃**

1. 任意节点用户说“暂存／先到这”→save_draft 后释放场景锁；
2. 草稿（含已上传材料与已填字段）保留 7 天。

**特殊情况6：敏感凭据密文**

1. 密钥 Key、AK/SK、API key 等敏感凭据全程密文，不明文回显、不朗读、不写入回复文本；
2. 仅通过密钥卡 icon 切换显示／隐藏或复制。

## 四、 完整示例对话

> 示例背景：信息科王工一次登记两类资源——放射的 PACS（业务系统·DICOM 对接）与一个 Qwen 大模型（模型资源·云端部署）。示例覆盖：
> 

> 1. 弹列表打招呼；
> 

> 2. 任意格式材料识别；
> 

> 3. 两类资源判定与回填；
> 

> 4. 缺失字段补全；
> 

> 5. 访问测试失败与修正；
> 

> 6. 自动访问测试通过；
> 

> 7. 资源注册成功。
> 

| **节点** | **角色** | **对话内容** | **系统动作 / 组件** |
| --- | --- | --- | --- |
| — | （进入） | 用户进入场景 | 触发自动发起 |
| N0 | 医小管 | （进场弹出「业务系统数据资源列表」与「模型资源列表」）你好！我是医小管。把资源信息文件发给我（支持 PDF、DOC、DOCX、XLSX、csv、jpg、jpeg、png、链接等任意文件格式），或文字、语音描述，我来帮你进行医院资源注册～ | 1. 弹两类资源列表；
2. 固定打招呼 |
| N1 | 用户 | （上传 资源清单.xlsx；语音："放射的 PACS 走 DICOM，再加一个 Qwen 大模型，云端部署"） | 1. 接收 1 文件＋1 语音；
