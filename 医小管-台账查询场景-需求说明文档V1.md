# 医小管-台账查询场景-需求说明文档

## 一、场景定位

| **项目** | **说明** |
| --- | --- |
| 场景名称 | 台账查询 |
| 操作完成方向（预设） | 1.自动开场：进入即由医小管固定欢迎「你好，我是医小管。台账相关问题都可以问我」，附 3 条推荐问句（生成今日《智能体管理情况报告》 / 查看某智能体 360 画像 / 查看当前告警情况），支持点击或语音/文字自由提问；
2.意图分流：每轮先识别意图并进入对应分支、总体/指标（四段回复 + 引导生成报告）、单智能体（呈现 360 画像 + 维度下钻）、超范围（固定友好兜底）、跑偏（固定拉回话术）；
3.收尾延展：加粗指标可点击下钻，报告可生成/编辑/导出并订阅日/周速读，360 画像支持按风险依据、资源明细、运行趋势、对比同类等维度继续下钻。 |
| 进入方式 | 1.意图路由：命中「看台账 / 台账查询 / 智能体家底 / 查看某智能体画像 / 告警情况」等口令；
2.快捷入口：统一对话页常驻【台账查询】；
3.从统一台账中心页、告警通知、报告订阅链接等相关入口进入；
4.无论何种进入方式，进入即由医小管自动打招呼、主动展示固定欢迎语与 3 条推荐问句，用户无需先开口。 |
| 完成标志 | 本场景为持续问答型，以「单轮闭环」为完成标志：
1.本轮已按意图给出对应产出（意图一：直答 + 口径 + 下钻 + 报告引导；意图二：定位回述 + 360 画像 + 维度下钻；超范围：固定兜底话术；跑偏：固定拉回话术）；
2.已提供下一步动作入口（下钻链接、生成/编辑/导出报告、画像模块跳转、订阅速读等）；
3.用户可持续追问，直至主动离开场景。 |
| 产出物 | 1.指标/总体直答内容（数值 + 统计口径 + 时间范围 + 可点击下钻入口）；
2.单智能体 360 画像视图（实体信息、关联资源拓扑、准入评测、运行监测，敏感字段密文）；
3.图文《智能体管理情况报告》（规模与分布 / 运行与调用 / 异常告警与故障 / 风险分级 / 准入评测进度五模块，图文并茂，可在线编辑、批注、导出，可附日/周速读订阅）；
4.超范围场景的固定友好提示与可答范围引导。 |
| 预计步数 / 时长 | 1.单轮问答约 10–30 秒；
2.一次完整会话通常 1–5 轮，可根据用户追问不限轮次持续进行。 |
| 数据范围与权限 | 1.信息科管理员＝全院；
2.科室用户＝本科室；
3.作答、图表、下钻均严格限权限范围；
4.API key 等敏感字段一律密文；
5.所有指标/清单/画像/报告数据均来自台账聚合与各中心工具真实返回，医小管只呈现与解读、不臆造；
6.无数据 / 超范围如实说明。 |

## 二、任务轨道（状态机节点表）

> 本场景为意图路由型（非顺序表单）：N0 自动开场 → N1 意图识别，按结果分流至 N2a 总体/指标（含 N2a-R 报告生成）、N2b 单智能体 360 画像、N2c 超范围兜底、N2d 跑偏拉回；分支闭环后回到接收态继续接新提问。多模态、放弃、修改、跑偏等通用规则引用《平台通用规范 v1》。
> 

| **节点** | **名称** | **动作（要点）** | **完成条件** |
| --- | --- | --- | --- |
| N0 | 场景进入·自动打招呼＋推荐问句 | 1.固定开场：「你好，我是医小管。台账相关问题都可以问我」；
2.下方展示 3 条推荐问句气泡：（1）帮我生成一份今日的全院/本科室智能体管理情况报告；（2）我想要查看【某智能体】的 360 画像；（3）目前智能体的告警情况……；
3.用户可点击一键发起，或语音/文字自由提问 | 1.医小管已展示固定欢迎语与 3 条推荐问句；
2.用户已通过点击推荐问句或自由提问给出首轮输入 |
| N1 | 意图识别 | 每轮先做意图判定：
1.含具体智能体（名称/编号/明确指代）→ 意图二；
2.含总体词或某总览指标（数量/科室覆盖率/总调用量/异常告警次数/正常运行率/每月新增/科室分布/诊疗环节分布/来源分布/风险分级）且无特定智能体 → 意图一；
3.台账相关但平台无此信息（未采集字段/无数据源/外部或未来预测）→ 超范围兜底；
4.两可/指代不明 → 先澄清一句后归类；
5.与台账完全无关 → 跑偏拉回 | 已产出明确意图标签（overview_metric / single_agent / out_of_scope / clarify / offtopic），并进入对应分支节点 |
| N2a | 意图一 · 总体/指标回复 | 四段式回复：
1.直答数值/结论（取自台账聚合真实数据，总体问法给多指标概况）；
2.标注统计口径与时间范围（如「已纳管；截至今日」「今日/本周/本月分别为…」）；
3.给下钻/分流（加粗指标可点击分流到对应列表，无卡片）；
4.主动引导「需要我据此生成《智能体管理情况报告》吗？图文并茂、可编辑导出」 | 1.已输出直答 + 口径 + 下钻入口 + 报告生成引导；
2.用户可继续追问、下钻或触发报告生成 |
| N2a-R | 报告生成分支 | 1.触发时机：（1）点击推荐问句①；（2）指标问答中同意生成报告；（3）直接说「生成管理情况报告 / 要向领导汇报」；
2.范围按角色默认全院（管理员）或本科室（科室用户），周期默认今日/近30天，可一句话调整（「改成本周」「只看放射科」）；
3.调用 summarize_ledger → 生成图文报告（规模与分布 / 运行与调用 / 异常告警与故障 / 风险分级 / 准入评测进度）→ 进入编辑页在线编辑/批注（自动存草稿）→ 一键导出为便于汇报的格式；
4.可附订阅台账速读（日/周） | 1.《智能体管理情况报告》草稿已生成并进入编辑页；
2.已提供导出与速读订阅入口 |
| N2b | 意图二 · 单智能体 360 画像 | 1. 定位与回述：按名称/编号/指代定位并回述确认（重名或指代不明先列候选卡请用户确认）；若为推荐问句②但未指名，追问「请问看哪个智能体？可报名称或编号」；
2. 呈现 360 画像：实体信息（所属科室、诊疗环节、功能描述、来源、供应商、接入方式；API key 密文）、关联资源拓扑（对接的院内系统/资源，异常连接醒目标注）、准入评测（总分/各安全维度/趋势）、运行监测（近 30 天调用量、成功率、平均时延、在线状态与趋势）；
3. 按维度下钻：顺用户问法直达对应模块（问「为什么高风险」→风险依据+处置；问「对接哪些系统」→资源拓扑；问「用得怎样」→运行监测）；
4. 延伸引导：末尾主动引导「需要看完整详情、风险依据，或对比同类智能体吗？」 | 1.已完成智能体定位与回述、已呈现 360 画像并按用户问法直达相应模块；
2.已提供延伸下钻入口 |
| N2c | 超范围兜底 | 1.使用固定不可改写话术「问题超出平台现有信息，我们将持续优化完善哟~」；
2.如有相近可答数据，补一句「目前我能提供的是……」引导到可答范围；
3.不臆测、不给无依据结论 | 1.固定兜底话术已发出；
2.如有相近可答数据，已给出可答范围引导 |
| N2d | 跑偏拉回 | 1.使用固定话术「我们先看台账，稍后再为您解决此问题」，随后复述可继续的方向（「台账这边还需要看什么？」）；
2.不切换场景、不展开解答；
3.平台内可解决请求记入待办，会话收尾后跟进；
4.越权/敏感请求按权限拒绝、密钥密文，不外泄 | 1.固定拉回话术已发出；
2.用户回到台账话题或选择离开 |
| N3 | 会话延续 / 收尾 | 1.单轮闭环后回到接收态：继续等待用户追问（下钻、切换意图、生成报告、查看画像等）；
2.用户主动离开或长时间无输入即结束场景 | 用户主动结束会话，或已提供全部下一步动作入口 |

## 三、字段清单

> 本表字段为查询对象（总览指标、画像维度、报告要素）与产物，不含表单收集；具体规则见「规则」列。
> 

| **字段 / 指标** | **来源** | **类型** | **规则（差异化）** |
| --- | --- | --- | --- |
| 智能体数量 | 台账聚合 | 整数 | 1.已审核纳管的智能体总数；
2.按角色权限（全院/本科室）过滤 |
| 科室覆盖率 | 台账聚合 | 百分比 | 1.已部署科室数 / 医院总科室数；
2.标注分子分母口径 |
| 总调用量 | 运行监测 | 整数（日/周/月） | 1.被调用次数，支持今日/本周/本月三种周期；
2.口径注明「接入运行后产生」 |
| 异常告警次数 | 运行监测 | 整数（日/周/月） | 1.告警总次数；
2.支持今日/本周/本月；
3.点击「告警 X 次」下钻至清单 |
| 正常运行率 | 运行监测 | 百分比 | 1.正常运行实例数 / 总实例数；
2.标注统计周期 |
| 每月新增纳管数量 | 台账聚合 | 整数序列 | 近 12 个月新增趋势，可按周/月/季度切换 |
| 科室分布 | 台账聚合 | 结构化列表 | 各科室数量与占比，按多→少排序 |
| 诊疗环节分布 | 台账聚合 | 结构化列表 | 9 类诊疗环节（导诊分诊/预问诊/预约挂号/辅助检查/辅助诊断/辅助治疗/住院/手术/其他）数量与占比 |
| 来源分布 | 台账聚合 | 结构化列表 | 自研 / 第三方 / 合作研发数量与占比 |
| 风险分级 | 台账聚合 | 结构化列表 | 1.高度关注 / 中度关注 / 一般关注数量与占比（初步/复核）；
2.只呈现不改判 |
| 智能体标识 | 对话获取 | 短文本 / 枚举 | 1.意图二定位依据：名称、编号（如 0503-0001）、或明确指代；
2.重名/相似列候选确认；
3.指代不明先追问 |
| 360 画像 · 实体信息 | 台账聚合 | 结构化对象 | 1.所属科室、诊疗环节、功能描述、来源、供应商、接入方式及分支字段；
2.API key 密文 |
| 360 画像 · 关联资源拓扑 | 资源中心 | 拓扑图 | 1.对接的院内系统/资源；
2.异常连接醒目标注；
3.可下钻明细 |
| 360 画像 · 准入评测 | 准入评测中心 | 结构化对象 | 评测结果总分、各安全维度得分、评测趋势 |
| 360 画像 · 运行监测 | 运行监测 | 结构化对象 + 趋势 | 1.近 30 天调用量、成功率、平均时延、在线/异常状态与趋势；
2.无数据如实标注 |
| 报告范围 | 系统推断 + 对话获取 | 枚举 | 1.按角色默认全院（管理员）/ 本科室（科室用户）；
2.可一句话调整（如「只看放射科」） |
| 报告周期 | 系统推断 + 对话获取 | 枚举 | 1.默认今日 / 近 30 天；
2.可一句话调整（如「改成本周」） |
| 《智能体管理情况报告》 | 动作产出 | 图文文档 | 1.五模块（规模与分布 / 运行与调用 / 异常告警与故障 / 风险分级 / 准入评测进度），每模块配图 + 文字综述；
2.生成后进编辑页可编辑/批注/导出/订阅日周速读 |
| 意图标签 | 系统识别 | 枚举 | 1.overview_metric / single_agent / out_of_scope / clarify / offtopic；
2.每轮回复必含 |

**校验失败与边界文案**：

- **超范围（台账相关但平台无此信息）**：固定文案「问题超出平台现有信息，我们将持续优化完善哟~」，可补「目前我能提供的是……」引导到可答范围。
- **跑偏（与台账无关）**：固定文案「我们先看台账，稍后再为您解决此问题」+ 复述可继续方向。
- **越权/敏感**：按权限拒绝；API key 等敏感字段一律密文（），不外泄。
- **指代不明**：追问「请问看哪个智能体？可报名称或编号」；重名/相似列候选卡确认。
