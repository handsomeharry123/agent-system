# 医小管-台账查询场景-需求说明文档

## 一、场景定位

| **项目** | **说明** |
| --- | --- |
| 场景名称 | 台账查询 |
| 操作完成方向（预设） | 1.自动开场：进入即由医小管固定欢迎「你好，我是医小管。台账相关问题都可以问我」，附 3 条推荐问句（生成今日《智能体管理情况报告》 / 查看某智能体 360 画像 / 查看当前告警情况），支持点击或语音/文字自由提问；
2.意图分流：每轮先识别意图并进入对应分支、总体/指标（四段回复 + 引导生成报告）、单智能体（呈现 360 画像 + 维度下钻）、超范围（固定友好兜底）、跑偏（固定拉回话术）；
3.收尾延展：加粗指标可点击下钻，报告可生成/编辑/导出并订阅日/周速读，360 画像支持按风险依据、资源明细、运行趋势、对比同类等维度继续下钻。 |
| 进入方式 | 1.意图路由：命中「看台账 / 台账查询 / 智能体家底 / 查看某智能体画像 / 告警情况」等口令；
2.快捷入口：统一对话页常驻【台账查询】；
3.从统一台账中心页、告警通知、报告订阅链接等相关入口进入；
4.无论何种进入方式，进入即由医小管自动打招呼、主动展示固定欢迎语与 3 条推荐问句，用户无需先开口。 |
| 完成标志 | 本场景为持续问答型，以「单轮闭环」为完成标志：
1.本轮已按意图给出对应产出（意图一：直答 + 口径 + 下钻 + 报告引导；意图二：定位回述 + 360 画像 + 维度下钻；超范围：固定兜底话术；跑偏：固定拉回话术）；
2.已提供下一步动作入口（下钻链接、生成/编辑/导出报告、画像模块跳转、订阅速读等）；
3.用户可持续追问，直至主动离开场景。 |
| 产出物 | 1.指标/总体直答内容（数值 + 统计口径 + 时间范围 + 可点击下钻入口）；
2.单智能体 360 画像视图（实体信息、关联资源拓扑、准入评测、运行监测，敏感字段密文）；
3.图文《智能体管理情况报告》（规模与分布 / 运行与调用 / 异常告警与故障 / 风险分级 / 准入评测进度五模块，图文并茂，可在线编辑、批注、导出，可附日/周速读订阅）；
4.超范围场景的固定友好提示与可答范围引导。 |
| 预计步数 / 时长 | 1.单轮问答约 10–30 秒；
2.一次完整会话通常 1–5 轮，可根据用户追问不限轮次持续进行。 |
| 数据范围与权限 | 1.信息科管理员＝全院；
2.科室用户＝本科室；
3.作答、图表、下钻均严格限权限范围；
4.API key 等敏感字段一律密文；
5.所有指标/清单/画像/报告数据均来自台账聚合与各中心工具真实返回，医小管只呈现与解读、不臆造；
6.无数据 / 超范围如实说明。 |

## 二、任务轨道（状态机节点表）

> 本场景为意图路由型（非顺序表单）：N0 自动开场 → N1 意图识别，按结果分流至 N2a 总体/指标（含 N2a-R 报告生成）、N2b 单智能体 360 画像、N2c 超范围兜底、N2d 跑偏拉回；分支闭环后回到接收态继续接新提问。多模态、放弃、修改、跑偏等通用规则引用《平台通用规范 v1》。
> 

### 1. 开启对话入口说明

入口方式：

**方式一：首页对话框直接发送消息（文字/语音），以意图命中自然进入台账查询；**

用户先发出的那条消息即为首轮输入，医小管直接进入意图识别并作答（见第二章），不再重复欢迎语。

用户可能出现的提问方式：

| **查询对象** | **用户可能的提问方式（示例）** | **命中指标/口径** |
| --- | --- | --- |
| 总体情况 | “整体情况怎么样”“全院智能体家底”“台账总览看看”“有什么要重点关注的” | 总览概况（多指标汇总＋关注项） |
| 智能体数量 | “一共多少个智能体”“纳管了多少” | 已审核纳管的智能体总数 |
| 科室覆盖率 | “多少科室在用”“科室覆盖率多少” | 已部署科室数 / 医院总科室数 |
| 总调用量 | “调用量多少”“今天/本周/本月被调用多少次” | 被调用次数（含日/周/月） |
| 异常告警次数 | “有多少告警”“今天报了多少异常”“告警多不多” | 告警总次数（含日/周/月） |
| 正常运行率 | “运行率多少”“在线率/稳不稳定” | 正常运行实例数 / 总实例数（%） |
| 每月新增纳管数量 | “这个月新增几个”“新增趋势” | 近 12 个月新增趋势（可按周/月/季度） |
| 科室分布 | “哪个科室最多”“按科室分布” | 各科室数量与占比（多→少） |
| 诊疗环节分布 | “诊疗环节分布”“哪个环节多” | 各诊疗环节数量与占比 |
| 来源分布 | “自研和第三方各多少”“来源分布” | 自研/第三方/合作研发数量与占比 |
| 风险分级 | “高度关注几个”“风险分布”“有多少高风险” | 高度/中度/一般关注数量与占比（初步/复核） |

**方式二：首页点击业务标签「台账查询」快捷开启对话；**

如果是通过方式二点击业务标签「台账查询」发起对话，由智能体主动发送消息，对话气泡下方展示 3 条推荐问句：

「你好，我是医小管。台账相关问题都可以问我」

（1）帮我生成一份今日的全院/本科室智能体管理情况报告；

（2）我想要查看【某智能体】的 360 画像；

（3）目前智能体的告警情况；

**两种方式均在左侧「历史会话」新建并进入对应的对话窗口，后续追问在同一窗口内持续进行。用户可点击任意推进问话发起对话，或语音/文字自由提问。**

### 2. 意图识别

医小管对用户每轮输入先做意图判定，归入以下三类之一，并分流至对应子章节：

**意图一：查看总体情况 / 某指标（对应第 3 子章 · 意图一）**

- 判定依据：含「总体 / 家底 / 整体情况」等总体词，或命中总览指标（智能体数量 / 科室覆盖率 / 总调用量 / 异常告警次数 / 正常运行率 / 每月新增 / 科室分布 / 诊疗环节分布 / 来源分布 / 风险分级），且未指向特定智能体；
- 后续处理：转 N2a 总体/指标回复（含 N2a-R 报告生成分支）。

**意图二：查看单智能体信息（对应第 3 子章 · 意图二）**

- 判定依据：含具体智能体（名称、编号如 0503-0001、或明确指代）；
- 后续处理：转 N2b 单智能体 360 画像。

**特殊情况（对应第 5 子章 · 特殊情况说明）**

- 超范围：台账相关但平台无此信息（未采集字段 / 无数据源 / 外部或未来预测）→ 转 N2c 超范围兜底；
- 跑偏：与台账完全无关的话题 → 转 N2d 跑偏拉回；
- 两可 / 指代不明：先追问一句澄清后重新归入上述任一类。

**完成条件**：已产出明确意图，并进入对应分支节点。

### 3. 执行对应意图

根据 N1 意图识别结果，分两条主线执行：意图一走「总体指标回答 + 是否生成报告」，意图二走「呈现单个智能体 360 画像」。

**意图一 · 总体指标回答（含是否生成报告）**

> 以「四段式」直接回答用户的总体/指标问题，并在末尾主动引导生成《智能体管理情况报告》；用户确认后进入报告生成分支。
> 

指标回答：

1. 指标数据回答：给出指标数值/结论（总体问法给多指标概览，数据取自台账聚合真实数据），并标注统计口径与时间范围（如「已纳管；截至今日」「今日/本周/本月分别为…」）；加粗指标可点击，直接分流到对应列表；
2. 引导报告：主动追问「需要我据此生成《智能体管理情况报告》吗？」。

报告生成分支（用户确认后触发）：

1. 触发时机：① 点击推荐问句①；② 上一轮指标问答中用户同意生成；③ 用户直接说「生成管理情况报告 / 要向领导汇报」；
2. 范围与周期：范围按角色默认全院报告（信息科管理员）/ 本科室报告（科室用户）；周期默认今日，可一句话调整（如「改成本周、本月」）；
3. 生成：调用台账中心报告模块生成（规模与分布 / 运行与调用 / 异常告警与故障 / 风险分级 / 准入评测进度）；
4. 编辑与导出：点击生成的报告进入报告详情页，支持编辑、导出。

**意图二 · 呈现单个智能体 360 画像**

> 先定位到目标智能体（必要时列候选卡请用户确认），再呈现 360 画像四模块，并按用户问法直达对应维度下钻。
> 

执行步骤：

1. 定位智能体：按名称/编号/指代定位并回述确认；重名或指代不明则列候选卡请用户选择；未指名（如推荐问句②）则追问「请问看哪个智能体？可报名称或编号」；
2. 呈现 360 画像：用户可点击链接，打开台账列表360画像页面
3. 维度下钻与延伸引导：根据用户追问直接跳转到对应模块（如「为什么高风险」→ 风险依据+处置；「对接哪些系统」→ 资源拓扑；「用得怎样」→ 运行监测）；回复末尾主动追问「是否查看完整详情、风险依据，或对比同类智能体？」。

### 4. 对话延续及收尾

1. 单轮回复完成后，医小管保持接收状态，等待用户继续追问（如下钻、切换意图、生成报告、查看画像等）；
2. 若用户主动关闭对话或长时间无输入，本场景自动结束。

### 5. 特殊情况说明

**特殊情况1：超范围兜底**

情况定义：**用户提问虽与台账相关，但超出平台现有可获取信息范围（如平台未采集的字段、无对应数据源的问法、外部/未来预测类问题）；此时医小管不臆造、不硬答，统一使用固定友好提示。**

处理方式：

1. 发送固定话术（不可改写）：「问题超出平台现有信息，我们将持续优化完善哟~」；
2. 若有相近的可答数据，补一句「目前我能提供的是……」，引导至可答范围；
3. 全程不猜测、不输出无依据的结论。

**特殊情况2：跑偏拉回**

处理方式：

1. 发送固定话术「我们先看台账，稍后再为您解决此问题」，随后追问可继续的方向（如「台账这边还需要看什么？」）；
2. 不切换场景、不对无关话题展开解答；
3. 平台内可解决的请求记入待办，本轮会话收尾后再跟进；
4. 越权或敏感请求按权限拒绝，密钥统一密文处理，不对外泄露。
