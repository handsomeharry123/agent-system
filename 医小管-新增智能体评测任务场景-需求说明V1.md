# 医小管-新增智能体评测任务场景-需求说明

## 一、场景定位

| **项** | **说明** |
| --- | --- |
| 场景名称 | 新建评测 |
| 操作完成方向（预设） | 1. 用户（信息科管理员）进入即默认开始 新建智能体安全评测任务；
2. 沿固定任务轨道 N0→N(末) 收集/执行 2 项事项（选定智能体、各维度测试样本数档位）；
3. 完成标志为 评测任务新建成功并自动开始执行、进度实时可见、评测完成后结果回填台账；
4. 收尾后仅开放动作：① 查看评测进度 ② 查看评测结果详情 ③ 新建下一个评测 |
| 进入方式 | 1. 意图路由：命中「新建评测／给某智能体做评测／有哪些待评测」等口令；
2. 快捷入口：对话页常驻【新建评测】（仅管理员可见）；
3. 从 待评测队列/准入评测 页进入。无论何种方式，进入即由医小管自动打招呼、主动发起 |
| 完成标志 | 1. 评测任务新建成功并自动执行；
2. 总进度与五维度进度实时呈现；
3. 评测完成后总分、各维度得分、趋势回填智能体台账「评测结果信息」 |
| 产出物 | 1. 一条（或多条）智能体安全评测任务记录；
2. 各维度进度与得分、总分及趋势；
3. 回填至智能体台账「评测结果信息」的评测报告 |
| 预计步数 / 时长 | 1. 约 4 步（选定智能体→选样本数档位→确认→自动开始→完成回填）；
2. 交互耗时约 2–3 分钟（不含评测引擎执行时长） |

## 二、任务轨道

> **场景类型**：意图路由型任务——入场后按管理员每轮输入的意图分流至对应处理（区别于"顺序表单型"）。
**主流程**：自动开场·呈报待评测队列 → 意图识别 → 采集目标智能体与各维度样本数档位 → 配置确认并新建评测任务 → 自动开始评测·实时呈报进度 → 完成回填台账与收尾。
> 

### 1. 开启对话入口

入口方式：

**（1）方式一：首页对话框直接发送消息（文字/语音），以意图命中自然进入新建评测；**

管理员说"新建评测 / 给某智能体做评测 / 有哪些待评测"等命中口令，权限校验通过后直接进入场景，由医小管自动打招呼并呈报待评测队列。

**（2）方式二：首页点击业务标签「新建评测」快捷开启对话（仅管理员可见）；**

如果是通过方式二点击业务标签「新建评测」发起对话，由医小管主动发送固定开场语：

「你好！我是医小管，我来帮您新建智能体安全评测任务（约 4 步、2–3 分钟）。目前待评测 X 个智能体，请问要给哪个、选哪档评测？（快速 30% / 标准 60% / 深度 100%）」

并在下方以对话消息呈现待评测清单：智能体编号、名称、版本、各维度样本数（未选显示"待选择"）。X 由 「查询待评测队列」 返回。

**（3）两种方式均在「历史会话」新建独立窗口并延续多轮对话；本场景仅对信息科管理员开放，其他角色触发时提示无权限。**

### 2. 意图识别

医小管对管理员每轮输入先做意图判定，归入以下类别之一，并分流至对应处理：

**意图一：配置采集（对应 3 执行对应意图 · 意图一）**

- 判定依据：管理员指定目标智能体（按编号/名称/指代，支持单个或批量）、指定评测档位（五维度使用同一档位）、对追问项作答、或在任一节点（点/说/打字）"修改××"某项配置；
- 后续处理：进入待评测队列呈报、智能体定位与只读信息带出、各维度样本数档位设置与修改流程。

**意图二：确认新建并自动执行评测（对应 3 执行对应意图 · 意图二）**

- 判定依据：配置齐备后，管理员以点击、语音、文字任一方式发出"确认新建/开始评测"意图；或期间查询进度（"进度到哪了"）、说"暂存/先存着"；
- 后续处理：进入配置确认 → 新建评测任务 → 自动开始评测与进度实时呈报 → 完成回填台账与收尾流程。

### 3. 执行对应意图

根据 2 意图识别的结果，分流至以下两条执行链路；分支内步骤按 MECE 原则互斥、依序推进。

**3.1 意图一：配置采集**

> 三步推进：① 定位目标智能体 → ② 设置各维度样本数档位 → ③ 修改与循环补全。管理员仅决策两项——目标智能体、评测档位（五维度统一）；其余只读字段（编号/名称/版本/评测标准/评测维度）系统自动带出，不询问。
> 

**A. 待评测队列呈报与目标智能体定位**

1. **清单呈报**：调用「查询待评测队列」获取待评测清单，播报数量 X 并以对话消息呈现每个智能体的编号（“科室编号-准入顺序号”，如 0503-0001）、名称、版本、各维度样本数（未选显示“待选择”）；
2. **智能体定位**：依管理员指令（编号/名称/指代，如“给第一个”“给 XNK-0001”“这批都评一下”）定位目标，支持单个或批量；重名或指代不明则列候选请管理员确认；
3. **补充固定项**：定位后同步呈现固定评测标准《智能体安全评测规范》与固定五维度（输入安全/输出安全/行为安全/数据安全/工具安全）。

**B. 评测档位设置**

1. **档位定义**：三档可选——快速评测（抽题 30%）／标准评测（60%）／深度评测（100%）；档位越高覆盖越全、耗时越长，需在风险与效率间权衡；
2. **统一档位**：一次评测中五维度使用同一档位（如“五维度都用标准”）；本版本不支持分维度分别设置；
3. **决策辅助**：依风险等级给出建议（高度关注建议深度、一般关注建议标准/快速），最终由管理员决定。

**C. 修改（××）与循环补全**

1. 任一节点管理员说/点/打字"改档位 / 换智能体"等 → 定位目标节点重配置后回到中断处；
2. 循环直至目标智能体与五维度档位全部确定，方可推进至意图二。

**3.2 意图二：确认新建与自动执行评测**

> **主流程（四步）**：① 配置确认 → ② 新建任务 → ③ 自动开始评测与进度呈报 → ④ 完成回填台账与收尾。
**中途分流**：管理员说"修改××"先回意图一处理后返回本流程；说"暂存/先存着"则保存草稿不执行。
> 

**A. 配置确认呈现**

1. **呈现内容**：以配置确认消息逐项呈现——智能体只读信息（编号/名称/版本）＋固定评测标准《智能体安全评测规范》＋评测档位（五维度统一，对应抽题比例）；
2. **操作分流**："修改××"跳回 3.1 C 处理；"确认"推进至 B（点/说/打字三种等效，参见 4 对话延续·交互方式）。

**B. 新建评测任务**

管理员确认后调用「新建评测任务」，按选定档位比例从团体标准题集抽题（批量场景下多个智能体分别新建并关联各自抽题结果）。

**C. 自动开始评测与进度实时呈报**

1. **自动执行**：新建后自动执行「开始评测」（无需管理员再点【开始评测】）；批量任务并行或排队执行；
2. **进度呈报**：下方对话消息实时显示总进度＋各维度进度（已跑/总题、当前维度），批量任务分别播报；管理员随时问"进度到哪了"即时回报。

**D. 评测完成·结果回填与收尾**

1. **结果播报**：评测引擎产出结果后，医小管播报总分、五维度得分、（多次评测时）趋势；
2. **回填台账**：结果自动回填智能体台账「评测结果信息」，可作为风险分级与准入/复核参考；
3. **收尾提示**：给出三个动作入口——① 查看评测进度 ② 查看评测结果详情 ③ 新建下一个评测。

### 4. 对话延续及收尾

1. **交互方式**：每一步的确认／选择／修改／补填均支持手动操作（点击按钮/卡片编辑）、语音、文字三种等效方式，用户无需强制手动操作；
2. **对话延续**：单步完成后医小管保持接收状态，等待管理员继续操作（补填、修改、确认新建、暂存等）；评测任务创建并自动开始后，医小管持续以对话消息实时呈报进度，管理员可随时查询进度或继续建下一个评测；
3. **场景收尾**：评测完成、结果播报并回填台账后，本场景以三个收尾动作入口（① 查看评测进度 ② 查看评测结果详情 ③ 新建下一个评测）收束并释放场景锁；进行中的其他评测任务在后台继续，管理员可再进场对医小管说"看评测进度"查看。

### 5. 特殊情况说明

**特殊情况1：跑偏拉回**

情况定义：任一节点管理员询问内容跑偏（含闲聊、无关提问、请求其他平台操作），偏离当前评测任务新建。

处理方式：

1. **即时拉回**：发送固定话术（不可改写）"我们先来完成智能体评测，稍后再为您解决此问题"，随后复述当前节点事项；不切换场景、不展开无关话题；
2. **事后跟进**：平台内可解决的请求记入待办，本轮收尾后统一跟进。

**特殊情况2：越界请求**

情况定义：管理员要求跳过评测直接给分、篡改评测结果、修改评测标准或维度、或指定题库外结论等违反固定规则的请求。

处理方式：

1. **拒绝**：评测结果需由评测引擎依团体标准产生，医小管不臆造分数、不跳过评测、不修改标准与维度；越界内容不入配置、不执行、不写入回复结论；
2. **替代**：给出合规方案，如"要不要提高到深度评测重跑，或维持当前配置开始"。

**特殊情况3：任务生命周期变更（暂存 / 修改 / 中止）**

情况定义：管理员在评测流程中提出暂存、放弃，或修改配置（改样本数/换智能体等），涉及未开始、已开始或已完成的任务。

处理方式：

1. **暂存与放弃**：管理员说"暂存/先存着/不想继续" → 保存当前配置为草稿并释放场景锁，草稿保留 7 天；已通过 「新建评测任务」 启动的任务在后台继续执行，管理员可再进场对医小管说"看评测进度"查看；7 天内说"新建评测/继续评测"即可接着填；
2. **配置修改（按评测阶段分流）**：
    - 未开始（3.2 A 配置确认阶段）→ 直接跳回意图一相应节点重配置，回到本节点继续；
    - 已开始（3.2 C 进度呈报阶段）→ 明确说明重跑影响（题集将重新抽取、原进度作废），管理员确认后中止当前任务并按新配置新建；
    - 已完成（3.2 D 结果回填后）→ 建议以"新建下一个评测"重新发起，不覆盖已回填台账的历史结果与趋势。

**特殊情况4：批量任务并行**

情况定义：管理员一次要求为多个智能体建评测（如"这批都跑标准，那个高度关注的用深度"）。

处理方式：

1. **配置**：分别解析每个智能体的档位（可按风险等级给分别建议），一次汇总确认；
2. **执行**：分别调用 「新建评测任务」 新建、「开始评测」 并行或排队执行；
3. **呈报**：分别播报各任务进度与结果，管理员可就任一任务追问进度或建下一个。

**特殊情况5：权限与数据边界**

情况定义：非信息科管理员触发"新建评测"口令，或所询问的智能体不在管理员权限范围内。

处理方式：

1. **入口权限**：非管理员触发 → 权限校验失败，固定提示"新建评测仅对信息科管理员开放，暂无法为您办理"，不进入场景；
2. **数据权限**：越权智能体 → 不进入待评测清单、不定位、不作答，如实说明范围；
3. **回填权限**：结果回填台账遵循原有权限体系，不越权展示。

2. 含评测结果总分、各维度得分、（多次评测时）趋势；
3. 完成后自动回填智能体台账「评测结果信息」，可作为风险分级与准入/复核参考 |

特殊情况说明：

1. 只读字段（智能体编号、名称、版本、评测标准、评测维度）均取自台账或系统固定，本场景不询问、不允许修改；
2. 管理员说「改样本数／换智能体」→回对应节点重配（评测未开始前可改；已开始的需说明重跑影响）；
3. 越界请求（如要求跳过评测直接给分、篡改评测结果）一律拒绝；可建议提高档位重跑，但不得改分。
