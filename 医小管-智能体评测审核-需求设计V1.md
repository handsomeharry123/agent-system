# 医小管-智能体评测审核-需求设计

## 一、场景定位

| **项** | **说明** |
| --- | --- |
| 场景名称 | 评测审核 |
| 操作完成方向 | 1.管理员进入即由医小管自动打招呼、主动呈报"已评测待审队列"，无需管理员先发起；
2.沿固定任务轨道 N0→N4 逐条审核（进场呈报→选定并呈报评测结果→给出审核结论→填写具体说明→判定最终结果并生效同步）；
3.完成标志为清空待审队列或管理员暂停；
4.收尾后仅开放：① 继续下一条审核；② 查看审核记录；③ 查看批次汇总 |
| 进入方式 | 1.意图路由：命中「审核评测结果／准入审核／有哪些已评测待审」等口令，且权限校验通过；
2.快捷入口：对话页常驻【评测结果审核】（仅管理员可见）；
3.从"评测结果审核"页进入。
无论何种方式，进入即由医小管自动打招呼、主动发起 |
| 完成标志 | 1.每条已评测结果均产生一条审核结论并判定最终准入评测结果；
2.已落库（审核结论、具体说明、审核人、审核时间、最终结果）并完成状态流转；
3.已同步用户端并通知申请人；
4.或管理员暂停，未决项保持"待审核"状态、进度自动保存 |
| 产出物 | 1.每条审核记录：系统评测结果（依据）、审核结论、具体说明、审核人、审核时间、最终准入评测结果；
2.状态流转与用户端同步（审核通过=「具体说明」、退回修改=「退回原因说明」）；
3.批次汇总（准入 X／退回修改 Y／未处理 Z） |
| 预计步数 / 时长 | 1.单条约 4 步（呈报→结论→说明→最终结果确认）；
2.每条约 1–2 分钟；
3.批量审核总时长视队列长度而定 |

## 二、任务轨道（状态机节点表）

> 每节点独立一行；`动作`只写要点，通用规则不写；`出口判据`是节点被视为已完成的系统条件；`工具`只列本节点会触发的工具调用。
> 

| **节点** | **名称** | **动作（要点）** | **完成条件** |
| --- | --- | --- | --- |
| N0 | 进场与呈报 | 1.权限校验通过后固定开场：「你好！我是医小管。目前系统已评测 X 个智能体」；
2.调用 list_evaluated 拉取已评测待审队列，X 为返回数量；
3.以对话消息呈现清单：智能体编号、名称、系统评测结果（核心结论 准入／退回）及具体说明摘要 | 1.已评测清单已呈现；
2.待审数量已播报 |
| N1 | 选定与评测结果呈报 | 1.据管理员指令（按编号／名称／指代）选定一条并回述确认；
2.呈报智能体基本信息（只读：编号／名称／版本）；
3.呈报最新评测结果总览（系统核心结论 准入／退回 ＋ 具体说明，说明可由模型依评测规则生成）；
4.支持下钻各安全维度得分／评测明细 | 1.已选定唯一目标条目；
2.该条系统评测结果已呈报完成 |
| N2 | 审核结论（收指令） | 1.请管理员给出审核结论：审核通过／退回修改（单选、必填）；
2.管理员用自然语言下达，医小管解析为结论；
3.据结论联动具体说明填写要求（退回修改必填、审核通过选填） | 1.管理员已给出审核结论；
2.对应的具体说明填写要求已明确 |
| N3 | 具体说明 | 1.收集具体说明（≤500 字，实时字数）；
2.选"退回修改"必填（退回原因）；
3.选"审核通过"选填；
4.医小管回述并请管理员确认 | 1.具体说明满足所选结论的填写要求；
2.管理员已确认 |
| N4 | 判定最终结果·生效同步 | 1.据审核结论判定「最终准入评测结果」（映射：审核通过采纳系统结论——系统准入→准入、系统退回→退回；退回修改→退回修改）；
2.管理员确认后落库审核结论、具体说明、审核人、审核时间、最终结果；
3.按最终结果流转（准入可正式纳管／上线；退回或退回修改打回申请人整改或重评）；
4.将具体说明同步至用户端（审核通过=「具体说明」、退回修改=「退回原因说明」）并通知申请人；
5.下方显示编号、名称、最终评测结果及具体说明，随后回到 N1 处理下一条 | 1.最终结果已落定；
2.状态流转、用户端同步、申请人通知均已完成 |

## 三、字段清单

> 只列本场景相关字段；类型 / 校验优先复用《平台通用规范 v1》第九章基线；差异在「规则」列写明。
> 

| **字段** | **来源** | **类型** | **规则（差异化）** |
| --- | --- | --- | --- |
| 智能体编号 | 台账只读 | 短文本 | 1.取自智能体台账；
2.格式：科室编号-准入顺序号（如 0503-0001）；
3.只读、不可修改，对话中不涉及编辑 |
| 智能体名称 | 台账只读 | 短文本 | 1.取自智能体台账；
2.点击名称可跳转智能体详情页；
3.超过 10 字省略显示；
4.只读、不可修改 |
| 智能体版本 | 台账只读 | 短文本 | 1.取自智能体台账，如 1.0／1.1／2.0／2.1；
2.只读、不可修改 |
| 系统评测核心结论 | 系统产出 | 单选枚举 | 1.由评测引擎依评测规则产出：准入／退回；
2.作为审核依据；
3.不可篡改、不可编造，如有异议须走"退回修改"重评 |
| 系统评测具体说明 | 系统产出 | 长文本 | 1.由评测引擎依评测规则说明结论原因，可由模型生成；
2.支持下钻各安全维度得分／评测明细；
3.仅作为审核依据，不可篡改 |
| 审核结论 | 对话获取 | 单选枚举 | 1.单选：审核通过／退回修改，必填；
2.由管理员自然语言指令给出，医小管解析（不点单选、不填输入框）；
3.不同结论联动具体说明填写要求：退回修改必填、审核通过选填 |
| 具体说明 | 对话获取 | 长文本 | 1.≤500 字，实时字数校验；
2.选"退回修改"时必填（退回原因），未填不允许提交；
3.选"审核通过"时选填；
4.提交后同步用户端——退回修改显示为「退回原因说明」、审核通过显示为「具体说明」 |
| 审核人 | 系统生成 | 短文本 | 1.取当前登录管理员姓名；
2.对话中不涉及、自动落库 |
| 审核时间 | 系统生成 | 日期时间 | 1.系统自动记录审核提交时间；
2.格式 YYYY-MM-DD HH:MM |
| 最终准入评测结果 | 动作回填 | 单选枚举 | 1.由医小管据审核结论判定：审核通过→采纳系统结论（系统准入→准入、系统退回→退回）；退回修改→退回修改；
2.管理员确认后落库；
3.同一评测结果不重复出结论；
4.最终以管理员审核为准，系统评测结论仅为依据 |

特殊情况说明：

1.系统评测结论与分数为审核依据，管理员如有异议只能通过"退回修改"打回重评，不得直接修改系统评测结果或分数；

2.选择"退回修改"必须填写具体说明（退回原因），否则不允许提交；

3.管理员暂停时，未决项保持"待审核"状态，进度自动保存，随时可对医小管说"继续审核"恢复；同一条评测结果不重复出结论。
