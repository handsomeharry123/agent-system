# 医小管-监控信息查看-需求说明文档

## 一、场景定位

| **项目** | **说明** |
| --- | --- |
| 场景名称 | 监控信息查看 |
| 操作完成方向（预设） | 1. **进入即发起**：用户进入即由医小管自动打招呼并展示 3 条推荐问句，避免冷启动、降低开口成本；

2. 当前未处理告警清单（智能体编号、名称、告警内容，按告警级别高→低排序）；

## 二、任务轨道（状态机节点表）

> 本场景为**意图路由型**（非顺序表单）。整体流程：**进入 → 自动打招呼＋展示 3 条推荐问句 → 意图识别 → 分流到四条分支之一 → 分支闭环后回到接收态、等待下一轮提问**。
> 

四条分流分支：

**（1）意图一：总体情况 / 某监控指标**回复

**（2）意图二：查看告警情况，**弹出未处理告警清单，帮助用户逐条处置

**（3）超范围：使用**固定兜底话术回复

**（4）跑偏：使用**固定拉回话术回复

### 1. 开启对话入口说明

**1.1 入口方式：**

**方式一：首页对话框直接发送消息（文字/语音），以意图命中自然进入运行监控信息查看；**

用户先发出的那条消息即为首轮输入，医小管直接进入意图识别并作答（见「意图识别」小节），不再重复欢迎语。

**方式二：首页点击业务标签「监控信息查看」快捷开启对话；**

如果是通过方式二点击业务标签**「监控信息查看」**发起对话，由医小管主动发送消息，对话气泡下方展示 3 条推荐问句：

「你好，我是医小管。运行监控相关问题都可以问我」

（1）帮我生成一份今日的全院/本科室智能体运行监控情况报告？

（2）现在有哪些告警需要处理？

（3）今日智能体运行 token 成本消耗多少？

**1.2 两种方式均在左侧「历史会话」新建并进入对应的对话窗口，后续追问在同一窗口内持续进行。用户可点击任意推荐问句发起对话，或语音/文字自由提问。**

### 2. 意图识别

医小管对用户每轮输入先做意图判定，归入以下类别之一，并分流至对应子章节：

**2.1 意图一：查看总体情况 / 某监控指标（对应「执行对应意图 · 意图一」）**

- 判定依据：含总体/监控指标词且非告警，如「整体运行/运行概况/监控总览」或具体指标（任务成功率、中断率、调用次数、响应时间、token 成本等）；笼统「看看运行情况 / 现在稳不稳」也归入本意图入口；
- 后续处理：转「执行对应意图 · 意图一 · 指标/总体回复（含报告生成分支）」。

**2.2 意图二：查看告警情况（对应「执行对应意图 · 意图二」）**

- 判定依据：含告警/待处理/异常告警词，如「告警/报警/警报」「有哪些要处理」「报了什么异常」「触发了哪些阈值」「未处理告警」等，指向告警本身；
- 后续处理：转「执行对应意图 · 意图二 · 弹出未处理告警清单」。

**2.3 特殊情况（对应「特殊情况说明」）**

- 超范围：运行监控相关但平台无此信息（未采集指标 / 无对应数据源 / 外部或未来预测）→ 走超范围兜底；
- 跑偏：与运行监控完全无关的话题 → 走跑偏拉回；
- 两可 / 指代不明：先追问一句澄清后重新归入意图一或二。

### 3. 执行对应意图

根据意图识别结果，分两条主线执行：意图一走「四段式指标回复＋引导生成报告」，意图二走「弹出未处理告警清单＋逐条处置引导」。

**3.1 意图一 · 总体情况 / 某监控指标回复（含报告生成分支）**

> 以「四段式」直接回答用户的总体/监控指标问题，并在末尾主动引导生成《智能体运行监控情况报告》；用户确认后进入报告生成分支。
> 

指标回答：

1. 直答：给出用户问的数值/结论（取自各监控中心真实返回；总体问法给四维关键指标概况）；
2. 口径、告警与下钻：**口径**——标注统计口径与周期（如「今日 / 本周 / 本月」，成本类给日/周/月趋势）；**下钻**——加粗指标名称可点击进入监控明细 / 趋势；**告警对照**——若命中《告警规则配置》提示「已触发〔级别〕告警」，并以一句「这条已产生告警，要看未处理告警吗」引导切到意图二；
3. 引导报告：主动追问「需要我据此生成《智能体运行监控情况报告》吗？」。

报告生成分支（用户确认后触发）：

1. 触发时机：① 点击推荐问句①；② 上一轮指标问答中用户同意生成；③ 用户直接说「生成运行监控报告 / 要向领导汇报」；
2. 范围与周期：范围按角色默认全院（信息科管理员）/ 本科室（科室用户），可一句话调整（如「只看放射科」）；若用户已明确周期（今日/本周/本月/今年）则直接采用，未明确时先追问「日报/周报/月报/年报」；
3. 生成：调用 gen_monitor_report 生成图文并茂的《智能体运行监控情况报告》，按业务 / 状态 / 成本 / 安全四维组织，每维配图＋文字综述＋当前告警与关注项；
4. 编辑与导出：进入报告编辑页后可在线编辑/批注并一键导出，附可选订阅（日/周速读）。

**3.2 意图二 · 查看告警情况，弹出未处理告警清单**

> 直接弹出当前未处理告警清单，逐条呈现三项核心字段，并按级别由高到低排序，并引导逐条处置。
> 

执行步骤：

1. 清单弹出与三项核心字段：先回复「为您弹出当前未处理告警共 X 条（按级别高→低）：」，随后以表格逐条呈现三项核心字段：
    
    ① 智能体编号（如 0503-0001）；
    
    ② 智能体名称（如「糖尿病随访管理助手」）；
    
    ③ 告警内容（触发指标＋实测值＋所命中告警规则阈值/级别，如「任务执行成功率 82%，命中〔高〕级告警规则，阈值 85%」）；
    
2. 级别排序、权限与筛选处置引导：
    
    **（1）级别与排序：**告警级别按《告警规则配置》设定（低/中/高），清单默认高→低排序，高危置顶醒目；
    
    **（2）权限：**按角色权限过滤（信息科管理员＝全院，科室用户＝本科室），越权数据不呈现，密钥类信息密文；
    
    **（3）筛选：**支持按级别/维度/某智能体筛选（如「只看高危」「成本类告警」「XX 有没有告警」）；
    
    **（4）处置引导：**追问「需要我展开某条的触发详情或处置建议吗？可回复序号」，衔接查看详情 / 通知负责人 / 标记处理；
    
3. 无告警：若当前无未处理告警，如实回复「当前暂无未处理告警，运行平稳」，并引导「要不要看运行概况或生成监控报告」。

### 4. 对话延续及收尾

4.1 单轮回复完成后，医小管保持接收状态，等待用户继续追问（如下钻、切换意图、生成报告、逐条处置告警等）；

4.2 意图一与意图二可双向切换：

**（1）意图一 → 意图二**——当意图一（总体/指标回复）中所问指标已触发告警时，用户可点击提示「这条已产生告警，要看未处理告警吗」一键跳转到意图二查看未处理告警清单；

**（2）意图二 → 意图一**——在告警清单中可就某条告警追问相关指标数值或日/周/月趋势，切回意图一继续了解；

4.3 若用户主动关闭对话或长时间无输入，本场景自动结束。

### 5. 特殊情况说明

**5.1 特殊情况1：超范围兜底**

情况定义：**用户提问虽与运行监控相关，但超出平台现有可获取信息范围（如平台未采集的指标、无对应数据源的问法、外部/未来预测类问题）；此时医小管不臆造、不硬答，统一使用固定友好提示。**

处理方式：

1. **固定话术＋严守事实基线**：仅发送固定话术（不可改写）「问题超出平台现有信息，我们将持续优化完善哟~」，全程不猜测、不输出无依据的结论；
2. **引导至可答范围**：若有相近的可答数据，补一句「目前我能提供的是……」，引导用户转向平台已有能力的相关问题。

**5.2 特殊情况1：跑偏拉回**

处理方式：

1. **固定拉回话术＋守住场景边界**：发送固定话术「我们先看运行监控，稍后再为您解决此问题」，随后追问可继续的方向（如「运行监控这边还需要看什么？」）；全程不切换场景、不对无关话题展开解答；
2. **平台内请求记入待办**：跑偏话题若属平台内可解决的请求，统一记入待办，本轮会话收尾后再跟进；
3. **安全与权限底线**：越权或敏感请求一律按权限拒绝，密钥类信息统一密文处理（********），不对外泄露。

2. 用户已明确周期则直接采用、不再追问；

| 智能体名称 | 动作回填 | 短文本 | 1. 由告警清单接口返回（如「糖尿病随访管理助手」）；

3. 汇总当前告警与关注项；
4. 支持在编辑页在线编辑/批注，导出为便于汇报的格式（可附速读订阅）。 |
| 关注项 / 告警汇总 | 动作回填 | 列表 | 1. 报告中汇总当前已触发告警及关注指标；

3. 数据仅来自各监控中心工具真实返回。 |
| 播报时间 | 系统生成 | 日期时间 | 1. 自动记录本次问答/报告的时间；
2. 格式 YYYY-MM-DD HH:MM；

特殊情况说明：

1. 边界话术分明，勿混用：① 超范围（运行监控相关但平台无此信息）→「问题超出平台现有信息，我们将持续优化完善哟~」；② 跑偏（与运行监控完全无关）→「我们先看运行监控，稍后再为您解决此问题」；③ 越权/敏感 → 按权限拒绝、密钥保持密文（********）。
2. 数据事实性：所有指标、告警清单、成本用量、报告数据均来自各监控中心工具真实返回；医小管只呈现与解读、不臆造；无数据/超范围如实说明；告警级别与阈值按《告警规则配置》呈现、不改判。
3. 权限过滤：信息科管理员默认全院、科室用户默认本科室；作答、图表、告警清单、下钻均限权限范围；API key 等敏感字段密文。
