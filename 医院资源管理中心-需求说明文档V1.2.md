## 概述

### 功能架构

| 模块 | 页面 |
| --- | --- |
| 1. 资源管理 | 资源管理列表（全部资源 / 我的草稿 两个 Tab）、注册资源页 |
| 2. 资源申请 | 申请管理列表（全部申请 / 草稿 / 待审核 / 审核中 / 撤销修改 / 审核通过 / 退回修改 七个 Tab）、申请权限页、权限审批页、权限申请详情页 |

### 角色与操作权限映射

<aside>
👥

操作按钮角色对应表格口径：表格「管理员」对应**信息科管理员**，表格「用户」对应**科室管理员**；资源管理类操作（注册/编辑/删除资源、访问测试、提交、暂存）归信息科管理员。

</aside>

### 名词术语

| 缩写/术语 | 含义 |
| --- | --- |
| HL7 | 卫生信息交换标准（Health Level 7） |
| FHIR | 快速医疗互操作资源标准 |
| DICOM | 医学数字成像和通信标准 |
| MQ | 消息队列（Message Queue） |
| 对接方式 | 资源与平台之间的接入协议类型 |
| 智能体编号 | 科室编号-准入顺序号（如 XNK-0001） |

## 一、资源管理

### 1.1 全部资源tab（资源管理列表）

展示全部已注册资源，字段取自注册资源页。

**字段说明**

| 具体字段 | 说明 |
| --- | --- |
| 资源列表 | 取自注册资源页 |
| 资源负责人 | 取自注册资源页 |
| 联系方式 | 取自注册资源页 |
| 对接方式 | 取自注册资源页，HL7/FHIR/DICOM/数据库直连/MQ消息队列，不同接入方式动态展示对应子字段 |

**对接方式子字段（不同接入方式动态展示）**

| 对接方式 | 子字段 | 取值/说明 |
| --- | --- | --- |
| ①HL7协议 | HL7版本 | v2.x / v3 |
|  | 协议类型 | MLLP / HTTP Gateway |
|  | IP地址 | — |
|  | 端口号 | — |
| ②FHIR协议 | 接口协议类型 | HTTP/HTTPS/gRPC/WebService |
|  | URL地址 | — |
|  | 密钥Key | — |
| ③DICOM | DICOM名称 | — |
|  | DICOM IP地址 | — |
|  | DICOM端口 | — |
| ④数据库直连 | 数据库类型 | MySQL/Oracle/SQLServer/PostgreSQL |
|  | IP地址 | — |
|  | 端口 | — |
| ⑤MQ消息队列 | MQ类型 | Kafka / RabbitMQ / RocketMQ / ActiveMQ |
|  | Broker地址 | IP或域名 |
|  | 端口 | — |
|  | 认证方式 | AK / SK（Access Key）/SASL认证（Kafka常见） |

**操作与交互**

- 注册资源（信息科管理员）：点击进入资源信息编辑页面
- 编辑（信息科管理员）：点击进入注册资源编辑页面
- 删除（信息科管理员）：点击弹出确认对话框，确认后删除资源

### 1.2 我的草稿tab（注册资源草稿页）

存放暂存的资源注册记录，字段口径同注册资源页。

**字段说明**

| 具体字段 | 说明 |
| --- | --- |
| 资源列表 | 搜索选择，支持多选（同 1.3 注册资源页资源列表可选目录） |
| 资源负责人 | 必填；负责该资源管理的人员姓名；可从已有用户中选择或手动输入 |
| 联系方式 | 资源负责人的手机号码或座机号码 |
| 对接方式 | HL7/FHIR/DICOM/数据库直连/MQ消息队列，不同接入方式动态展示对应子字段（同 1.1 子字段定义） |

**操作与交互**

- 编辑（信息科管理员）：点击进入注册资源编辑页面
- 删除（信息科管理员）：点击弹出确认对话框，确认后删除资源

### 1.3 注册资源页

用于新增/编辑资源注册信息。

**字段说明**

| 具体字段 | 说明 |
| --- | --- |
| 资源列表 | 搜索选择，支持多选（见下方资源列表可选目录） |
| 资源负责人 | 必填；负责该资源管理的人员姓名；可从已有用户中选择或手动输入 |
| 联系方式 | 资源负责人的手机号码或座机号码 |
| 对接方式 | HL7/FHIR/DICOM/数据库直连/MQ消息队列，不同接入方式动态展示对应子字段（同 1.1 子字段定义） |

**操作与交互**

- 访问测试（信息科管理员）：点击后显示测试中动画，测试通过显示绿色成功标识，失败显示错误提示（包括错误代码和错误原因）
- 提交（信息科管理员）：校验所有必填字段，校验通过后执行提交动作；校验失败给出气泡提示说明
- 暂存（信息科管理员）：资源注册记录自动暂存至注册资源草稿页（我的草稿tab）

## 二、资源申请

### 2.1 申请管理列表

以 Tab 形式管理不同状态的申请记录，共 7 个 Tab。以下公共字段在 7 个 Tab 中均展示，各 Tab 另有特有字段、操作按钮与交互，下方按 Tab 分别说明。

**各 Tab 公共字段**

| 字段 | 说明 |
| --- | --- |
| 序号 | 系统自动生成，按申请提交时间倒序递增编号；支持翻页连续编号 1、2、3…… |
| 智能体编号 | 取自智能体注册表单自动生成的编号；科室编号-准入顺序号（如 XNK-0001） |
| 智能体名称 | 取自智能体注册表单；点击名称跳转"智能体详情页"；超出10字省略处理，悬浮展示完整名称 |
| 所属科室 | 取自智能体注册表单（科室代码+科室名称） |
| 诊疗环节 | 取自智能体注册表单选择项；（导诊分诊/预问诊/预约挂号/辅助检查/辅助诊断/辅助治疗/住院/手术/其他（填空）） |
| 功能描述 | 取自智能体注册表单"功能描述"字段；超出20字则省略号显示，悬浮展示完整内容 |
| 申请资源名称 | 取自申请人选择的资源；超出15字省略处理，悬浮展示完整名称 |

#### 2.1.1 全部申请tab

**字段**

- 公共字段：序号、智能体编号、智能体名称、所属科室、诊疗环节、功能描述、申请资源名称
- 特有字段：申请状态（草稿/待审核/审核中/审核通过/退回修改/撤销修改）

#### 2.1.2 草稿tab

**字段**

- 公共字段：序号、智能体编号、智能体名称、所属科室、诊疗环节、功能描述、申请资源名称
- 特有字段：最后编辑时间（YYYY-MM-DD HH:MM:SS）

**操作与交互**

- 查看详情（用户隔离，仅限自己创建的操作）：进入权限申请详情页（含完整申请信息+审批轨迹）
- 编辑（用户隔离，仅限自己创建的操作）：重新打开申请权限页面，回填已保存内容，可继续编辑提交
- 删除（用户隔离，仅限自己创建的操作）：弹出二次确认"确认删除该草稿？"，确认后从草稿列表移除

#### 2.1.3 待审核tab

**字段**

- 公共字段：序号、智能体编号、智能体名称、所属科室、诊疗环节、功能描述、申请资源名称
- 特有字段：提交审核时间（YYYY-MM-DD HH:MM:SS）

**操作与交互**

- 查看详情（信息科管理员、科室管理员）：进入权限申请详情页（含完整申请信息+审批轨迹）
- 审核（信息科管理员，仅管理员角色可见）：进入权限审批页，可选择"审核通过"或"退回修改"，需填写审核意见
- 撤销（科室管理员，仅申请人本人可见）：点击弹出"确认撤销该申请？"，确认后申请进入"撤销修改tab"

#### 2.1.4 审核中tab

**字段**

- 公共字段：序号、智能体编号、智能体名称、所属科室、诊疗环节、功能描述、申请资源名称
- 特有字段：提交审核时间（YYYY-MM-DD HH:MM:SS）

**操作与交互**

- 查看详情（信息科管理员、科室管理员）：进入权限申请详情页
- 撤销（科室管理员，仅申请人本人可见）：在管理员已开始审核但未出结果前可点击撤销，管理员审核完成后无法撤销

#### 2.1.5 撤销修改tab

**字段**

- 公共字段：序号、智能体编号、智能体名称、所属科室、诊疗环节、功能描述、申请资源名称
- 特有字段：撤销时间（YYYY-MM-DD HH:MM:SS）

**操作与交互**

- 查看详情（信息科管理员、科室管理员）：进入权限申请详情页。
- 编辑（科室管理员）：重新打开申请权限页面回填原内容，可修改后重新提交审核，提交后进入"待审核tab"
- 删除（科室管理员）：弹出二次确认，确认后永久删除该撤销记录

#### 2.1.6 审核通过tab

**字段**

- 公共字段：序号、智能体编号、智能体名称、所属科室、诊疗环节、功能描述、申请资源名称
- 特有字段：具体说明（取自管理员审核时填写的通过意见或备注说明，如有条件通过的备注；超出20字则省略号显示，悬浮展示完整内容）、审核通过时间（YYYY-MM-DD HH:MM:SS）

**操作与交互**

- 查看详情（信息科管理员、科室管理员）：进入权限申请详情页，展示完整申请信息、审核人、审核意见、生效资源权限明细；审核通过的资源权限自动同步至该台账中心模块智能体详情页"已对接资源列表"tab

#### 2.1.7 退回修改tab

**字段**

- 公共字段：序号、智能体编号、智能体名称、所属科室、诊疗环节、功能描述、申请资源名称
- 特有字段：退回原因说明（取自管理员审核时填写的退回原因；超出20字则省略号显示，悬浮展示完整原因）、退回时间（YYYY-MM-DD HH:MM:SS）

**操作与交互**

- 查看详情（信息科管理员、科室管理员）：进入权限申请详情页，可查看退回原因及完整审核轨迹
- 编辑（科室管理员）：根据退回原因修改申请内容，修改后重新提交进入"待审核tab"
- 自动归档：退回后超过30天未编辑提交，系统自动归档（不在列表展示，可在"全部申请tab"查询）

### 2.2 申请权限页

申请人发起资源权限申请的表单页。

**字段说明**

| 具体字段 | 说明 |
| --- | --- |
| 智能体编号 | 取自智能体注册表单自动生成的编号；科室编号-准入顺序号（如 XNK-0001） |
| 智能体名称 | 取自智能体注册表单；点击名称跳转"智能体详情页"；超出10字省略处理，悬浮展示完整名称 |
| 所属科室 | 取自智能体注册表单（科室代码+科室名称） |
| 诊疗环节 | 取自智能体注册表单选择项；（导诊分诊/预问诊/预约挂号/辅助检查/辅助诊断/辅助治疗/住院/手术/其他（填空）） |
| 功能描述 | 取自智能体注册表单"功能描述"字段；超出20字则省略号显示，悬浮展示完整内容 |
| 申请资源名称 | 申请人从已注册资源列表中进行选择；超出15字省略处理，悬浮展示完整名称 |

**操作与交互**

- 暂存（科室管理员）：校验必填字段，通过后保存当前填写的申请数据，保持在当前页面，供后续补充修改，可在草稿tab页查看暂存记录
- 访问测试（科室管理员）：发起对申请资源访问IP的连通性测试，测试通过显示绿色成功标识，失败显示错误提示（包括错误代码和错误原因）
- 提交（科室管理员）：校验所有必填字段，提交成功后跳转到"待审核页"，且状态变为"待审核"

### 2.3 权限审批页

<aside>
🔒

本页仅信息科管理员可操作。

</aside>

**2.3.1 权限申请信息**

| 字段 | 说明 |
| --- | --- |
| 智能体编号 | 取自智能体注册表单自动生成的编号；科室编号-准入顺序号（如 XNK-0001） |
| 智能体名称 | 取自智能体注册表单；点击名称跳转"智能体详情页"；超出10字省略处理，悬浮展示完整名称 |
| 所属科室 | 取自智能体注册表单（科室代码+科室名称） |
| 诊疗环节 | 取自智能体注册表单选择项；（导诊分诊/预问诊/预约挂号/辅助检查/辅助诊断/辅助治疗/住院/手术/其他（填空）） |
| 功能描述 | 取自智能体注册表单"功能描述"字段；超出20字则省略号显示，悬浮展示完整内容 |
| 申请资源名称 | 取自申请人选择的资源；超出15字省略处理，悬浮展示完整名称 |

**2.3.2 审核结论**

| 字段 | 说明 |
| --- | --- |
| 审核结论 | 管理员对本次权限申请的审核判定结果；单选项：审核通过/退回修改，必填，选择不同结论联动下方"具体说明"提示文案 |

**2.3.3 具体说明**

| 字段 | 说明 |
| --- | --- |
| 具体说明 | 管理员针对审核结论填写的详细意见、退回修改原因；多行文本输入框，字数限制500字并实时显示已输入字数，选择"退回修改"时必填，选择"审核通过"时选填，提交后同步至用户端"退回原因说明"或"具体说明"字段 |

**操作与交互**

- 编辑（信息科管理员）：可修改申请的部分信息；修改后需重新提交审批
- 审核通过（信息科管理员）：系统弹出确认提示框，确认后提交审批结果，更新权限状态
- 退回修改（信息科管理员）：系统带出备注输入框，填写退回原因，点击提交后将申请退回给用户修改完善

### 2.4 权限申请详情页

展示申请的完整信息与审批轨迹（只读）。

**2.4.1 权限申请信息**

| 字段 | 说明 |
