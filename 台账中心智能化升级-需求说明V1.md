# 台账中心智能化升级-需求说明

## 一、项目背景与目标

### 1.1 背景

统一台账中心把分散在**接入中心、准入评测沙盒、运行监控中心及院内数据系统**的信息，按智能体聚合成「一本账」。当前信息分散在各中心，管理员与科室用户需逐层翻页、跨系统查询才能掌握智能体全貌，家底不清、风险不及时、下钻低效。

### 1.2 目标

通过引入智能助手（Agent），围绕「聚合—汇报—分析」为两类角色提供差异化能力：

- **台账清晰**：以智能体为中心，跨中心聚合呈现基本、评测、告警、故障等信息，形成一本账。
- **风险通知**：智能体主动汇报关键态势与风险，点击气泡或对话窗口中的指标名称即可进入对应列表。
- **汇报报告**：基于台账数据一键生成图文并茂、可编辑导出的汇报报告，并支持速读订阅。

### 1.3 目标用户与角色

| **角色** | **诉求** | **关注点** |
| --- | --- | --- |
| 信息科管理员
（管理视角） | 全院智能体一本账，准入、运行、对接情况随时可查 | 家底清晰、风险及时、操作高效、按需下钻 |
| 科室管理员
（使用视角） | 用好本科室在用的智能体，了解其能力、使用效果与可用状态 | 能力清晰、用得顺手、效果可见、可用状态及时、按需下钻 |
| 智能助手（Agent） | 贯穿全链路的 AI 能力 | 信息聚合、态势汇报、自然语言问答、智能总结分析 |

## 二、需求范围总览

本次升级围绕台账中心的智能助手（Agent）能力展开。**信息科管理员（管理视角）**与**科室管理员（使用视角）**使用**相同的台账中心页面与功能**，差异仅在**数据范围**：同一页面下，信息科管理员可见**全院全部智能体**，科室管理员仅可见**本科室智能体**；所有欢迎语、指标分流、聚合作答、报告与 360 画像均按当前登录角色的授权范围呈现。

本次需求**按功能**划分为以下四个模块：

| **序号** | **功能模块** | **一句话描述** |
| --- | --- | --- |
| 1 | Agent 说明 | 全平台统一的 Agent 悬浮入口形象、视觉动画与对话窗口规范（与接入中心一致） |
| 2 | 台账中心各页面欢迎语 | Agent 进入各页面主动展示的欢迎语（态势速览 + 引导），按角色授权范围取数 |
| 3 | 报告总结 | 基于台账数据一键生成可编辑、可导出的汇报报告，并支持台账速读订阅 |
| 4 | 智能体 360 画像视图 | 以单个智能体为中心聚合基本 / 评测 / 告警 / 故障等信息，支持下钻明细 |

## 三、详细功能需求

本章统一描述两类角色的功能需求。**信息科管理员**与**科室管理员**使用相同的台账中心页面与功能，**差异仅在数据范围**：

| **角色** | **视角** | **数据范围** |
| --- | --- | --- |
| 信息科管理员 | 管理视角 | 可见**全院全部智能体** |
| 科室管理员 | 使用视角 | 仅可见**本科室智能体** |

**整体思路流程**：用户进入台账中心后的主线为——

1. **查看各页面 Agent 欢迎语**：进入台账中心任一页面，Agent 悬浮机器人主动弹出欢迎语（态势速览 + 引导按钮），可点击加粗指标名称分流或唤起对话问答；
2. **查看 / 生成报告**：在欢迎语引导下一键生成台账汇报报告，进入编辑页在线编辑、批注并导出；
3. **查看单个智能体的 360 画像视图**：从台账列表进入某智能体详情页，默认展示其 360 画像视图，聚合基本信息、关联资源拓扑、准入评测与运行监测并可下钻。

下文按「**Agent 说明 → 各页面欢迎语 → 报告总结 → 智能体 360 画像视图**」四个功能模块展开：

### 3.1 Agent 说明

台账中心 Agent 的形象、视觉动画与对话窗口字段定义均与 [接入中心智能化升级-需求说明](https://app.notion.com/p/777083fb617082ffba38814b9194ac0c?pvs=21) 保持一致，仅业务内容不同：台账端以态势汇报与台账问答（查询问答）为主，上传附件为可选能力。该规范对信息科管理员与科室管理员一致，**仅数据范围不同**（管理员全院、科室本科室）。

**入口 icon 说明：**

入口采用一个**全身立体、拟真的小机器人**形象，作为全局常驻的悬浮入口（**默认位于页面右下角，支持鼠标拖拽自由移动到页面任意位置**）。它不是一个平面图标，而是一个「有生命感、会陪伴用户」的智能助手，随使用进程给予情绪反馈与引导。

**视觉效果：**

- **造型**：**全身立体（3D / C4D 渲染质感）**的拟真小机器人，可站立、拟人比例，具备头部、躯干、双臂与双足；面部有表情（大眼睛 / 发光眼 + 胸前灯带），整体像一个可爱、可交流的桌面伙伴。
- **质感与配色**：金属 + 磨砂塑料质感，带圆润倒角与高光；配色与平台主色一致（科技蓝 + 高光），带柔和光晕 / 投影与地面接触阴影，强化立体悬浮感。
- **陪伴属性**：拟人化微表情与肢体语言（点头、招手、思考挠头、欢呼），会随使用进度 / 操作给予情绪反馈（成功比心 / 庆祝、异常安抚鼓励），形成「全程陪伴的智能助手」人设。
- **尺寸**：常态约 64×64px 立体站姿形象，hover 放大至约 72px；底部保留轻微地面阴影与点击热区。
- **位置与拖拽**：默认悬浮于页面右下角；支持鼠标按住机器人拖拽，自由移动到页面任意位置，松开即停靠并记忆位置供后续访问沿用；拖拽过程中略降透明度、跟随光标移动，此时不触发点击唤起，避免误操作。
- **状态角标**：右上角可叠加未读红点 / 数字角标，提示新消息或待办。

**动画说明**

| **状态** | **动画表现** |
| --- | --- |
| 常态待机 | 站立状态轻微「呼吸」起伏 + 重心小幅晃动（2–3s 循环），偶发眨眼 / 环顾四周 / 胸前灯带呼吸闪烁，像真机器人般灵动待命 |
| 主动气泡态势速览 | 进入台账总览 / 列表页时，机器人旁自动弹出指向它的态势速览气泡（参考各页面欢迎语说明）；气泡淡入上浮、停留数秒引导查看态势与分流，点击气泡或机器人展开完整对话窗口，超时未操作自动收起为待机态 |
| 悬浮 hover | 轻微跳起 / 上浮并转身朝向用户 + 招手 / 点头问候，光晕增强，提示可点击 |
| 唤起点击 | 机器人点头 / 挥手后，对话窗口从 icon 位置展开（缩放 + 透明度渐入，约 250ms ease-out） |
| 新消息 / 提示 | 原地跳动 / 挥手（bounce）+ 右上角红点放大闪烁，吸引注意 |
| 思考 / 识别中 | 头顶循环省略号或齿轮转动，眼睛转为加载态，表示 Agent 正在聚合 / 作答 |
| 情绪反馈 | 陪伴化反馈：查询成功 / 报告生成时比心 / 鼓掌 / 庆祝，出现告警 / 故障等风险时低头安抚并提示优先处理，增强陪伴感 |
| 收起 | 窗口收拢回机器人位置，其挥手 / 坐下后回到站立待机状态 |

**Agent对话窗口页面字段**：

| **字段** | **说明** |

| 唤起入口 | 全局常驻的悬浮机器人入口，默认位于页面右下角，支持鼠标拖拽移动到页面任意位置；点击唤起对话窗口，可随时呼出助手 |

### 3.3 报告总结

智能助手基于台账数据一键生成图文并茂、可编辑导出的汇报报告，并支持台账速读订阅（日 / 周）。**两类角色报告能力一致，仅口径与数据范围不同**：信息科管理员生成「全院智能体运行管理情况」（报告包含全院数据），科室管理员生成「本科室智能体运行情况报告」（报告包含本科室数据）。

- **【生成报告】**：一键生成图文报告，生成后进入报告编辑页在线编辑、导出；
- **【订阅报告】**：设置台账速读（日 / 周）订阅，定时推送至台账总览-历史报告（窗口中）。

**视觉与动效补充说明（用于 Demo 生成）：**

本模块摒弃原「素净」的单节点样式，升级为**具科技感的多层电流拓扑关系网**，以智能体为绝对主角。以下为 Demo 还原参考（配套原型见本页末尾嵌入的可交互 Demo）。

**① 整体风格与画布**

- 深色科技风：画布深蓝径向渐变（中心 `#0d2350` → 边缘 `#04091c`），主色蓝青 `#38e6ff` / `#2b6cff`。
- 背景层叠：淡科技网格（中心亮、四周渐隐）+ 中心径向辉光 + 缓慢旋转的雷达虚线环与同心导引圈，营造「监测大屏」氛围。
- 全模块发光描边、玻璃拟态卡片，弱化硬边框。

**② 拓扑结构：以智能体为中心，向外 3–5 层放射网**

| **层级** | **含义** | **示例节点** | **配色（由内向外渐变）** |
| --- | --- | --- | --- |
| L0 中心 | 当前智能体（主角） | 互联网医院智能问诊助手 | 蓝青高亮 `#38e6ff` |
| L1 业务域 | 资源大类 | 临床业务 / 医技检查 / 药事服务 / 知识智库 / 医保结算 / 患者服务 | `#7ff3ff` |
| L2 核心系统 | 院内系统 | HIS / EMR / LIS / PACS / 药品库 / 医保系统 … | `#4fb6ff` |
| L3 功能模块 | 系统内子模块 | 门诊管理 / 检验报告 / 药品目录 … | `#5e9fe8` |
| L4 数据资源 | 末端数据/接口 | 电子处方 / 报告明细 / 结算数据 … | `#a9c6ff` |
- 采用**环形放射（径向树）布局**：各层半径由内向外递增，子节点按其「叶子数量」占比自动分配角度，保证均匀不重叠。
- 层级数据可配置，支持展示 3–5 层（当前 Demo 展示 4 层）。

**③ 中心智能体（主角，可爱 + 动画）**

- 圆润机器人虚拟形象：发光大眼睛会**周期眨眼**、微笑嘴型、腮红，头顶天线 + **旋转螺旋桨**，整体呆萌。
- 动效：整体**呼吸式上下浮动**、外圈**光环反向旋转**、由中心向外一圈圈扩散的**雷达脉冲波**，强化「主角/心跳」感。
- 底部名称牌：智能体名称 + 编号（如 AskMed-INFO-3.0）+ 在线状态。

**④ 连线电流动效（通电 / 电流流动感）**

- 每条连线为发光贝塞尔曲线，沿路径持续流动**明亮光点**（主体自中心**向外发散**，另有少量光点**回流至中心**，表现数据双向流动），叠加虚线滚动。
- 悬停连线/节点时整条链路高亮加亮。

**⑤ 异常连接高亮**

- 异常连线与节点使用红色 `#ff5468`，并以**更快速的红色电流 + 闪烁**醒目提示，与正常蓝青连接形成强对比。
- 右上角常驻「已连接 N / 异常 M」计数角标（异常数带闪烁红点）。

**⑥ 节点样式与下钻交互**

- L1 / L2：胶囊卡片（图标 + 名称 + 发光描边）；L3 / L4：发光圆点 + 外侧横向标签（按角度自动左右对齐，保证可读）。
- 悬停：节点放大并弹出浮层，展示**资源负责人 / 对接方式 / 对接状态**。
- 点击：下钻至对应资源明细（异常节点下钻至异常详情）。

**⑦ 实现与兼容建议**

- 建议 SVG + CSS/SMIL 动画实现（电流光点用沿路径运动的光点），性能友好；
- 尊重系统「减弱动态效果（prefers-reduced-motion）」偏好；中文字体需带回退（如 PingFang SC / Microsoft YaHei / Noto Sans SC），避免缺字。
