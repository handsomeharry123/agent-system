# 接入中心智能化升级-需求说明

## 一、项目背景与目标

### 1.1 背景

接入中心是智能体提供方申请接入、管理员审核准入的核心枢纽。当前接入流程存在录入工作量大、信息易出错、连通排障耗时、接入进度不透明等问题，导致提供方与管理员之间频繁来回沟通，效率低下。

### 1.2 目标

通过引入智能助手（Agent），覆盖「填—审—测—察」接入全链路，实现：

- **降门槛**：以对话式、多模态方式自动填表，降低提供方录入成本。
- **提质量**：填写即审查、审核即预审，自动发现并纠正问题。
- **增效率**：自动连通测试与故障诊断，复用历史成功方案。
- **强闭环**：主动洞察接入进度并引导下一步动作，形成完整闭环。

### 1.3 目标用户与角色

| 角色 | 描述 | 核心诉求 |
| --- | --- | --- |
| 智能体提供方
（角色：科室管理员、信息科管理员） | 申请接入智能体的业务方 | 快速、低成本完成信息填报与连通验证 |
| 智能体管理方
（角色：信息科管理员） | 负责审核与准入决策的管理员 | 快速掌握审核态势，高效完成审核决策 |
| 智能助手（Agent） | 贯穿全链路的 AI 能力 | 预填、审查、测试、预审、洞察与引导 |

## 二、Agent 功能说明

> 说明贯穿接入全链路的智能助手（Agent）本体：入口形象、视觉与动画表现、对话浮层字段与通用交互能力。各页面欢迎语引导见「三、注册接入引导说明」。
> 

### 2.1 Agent 对话入口

**入口 icon 说明**

入口采用一个**全身立体、拟真的小机器人**形象，作为全局常驻的悬浮入口（**默认位于页面右下角，支持鼠标拖拽自由移动到页面任意位置**）。它不是一个平面图标，而是一个「有生命感、会陪伴用户」的接入小助手，随填报进程给予情绪反馈与引导。

**视觉效果**

- **造型**：**全身立体（3D / C4D 渲染质感）**的拟真小机器人，可站立、拟人比例，具备头部、躯干、双臂与双足；面部有表情（大眼睛 / 发光眼 + 胸前灯带），整体像一个可爱、可交流的桌面伙伴。
- **质感与配色**：金属 + 磨砂塑料质感，带圆润倒角与高光；配色与平台主色一致（科技蓝 + 高光），带柔和光晕 / 投影与地面接触阴影，强化立体悬浮感。
- **陆伴属性**：拟人化微表情与肢体语言（点头、招手、思考挠头、欢呼），会随接入进度 / 操作给予情绪反馈（成功比心 / 庆祝、失败安抚鼓励），形成「全程陪伴的接入小助手」人设。
- **尺寸**：常态约 64×64px 立体站姿形象，hover 放大至 约 72px；底部保留轻微地面阴影与点击热区。
- **位置与拖拽**：默认悬浮于页面右下角；支持鼠标按住机器人拖拽，自由移动到页面任意位置，松开即停靠并记忆位置供后续访问沿用；拖拽过程中略降透明度、跟随光标移动，此时不触发点击唤起，避免误操作。
- **状态徐标**：右上角可叠加未读红点 / 数字角标，提示新消息或待办。

| 关闭窗口按钮 | 位于标题栏右上角，点击收起对话窗口（不清空会话）；下次唤起可继续 |

| 关闭对话 | 点击标题栏关闭按钮收起窗口，不清空会话，下次唤起可继续 |

**数据范围**：对话问答与聚合作答按当前角色授权范围进行——信息科管理员覆盖全院数据，科室管理员自动限定在本科室授权范围内。

**欢迎语规则**

- 文案在对话窗口与机器人气泡同步展示，`X` 为动态填充的实际数据。
- 每条欢迎语气泡内附带快捷操作按钮，一键进入下一步；无权限时置灰。

**气泡按钮指向规则**

- **多记录列表页**（草稿 / 待审核 / 审核中 / 退回修改 / 撤销修改 / 审核通过等）：气泡按钮不直接操作单条记录，而是先展开迷你清单（按时间倒序取前 3–5 条，底部附「查看全部」回到对应 tab），编辑 / 删除 / 查看详情 / 撤销等记录级操作由清单中每条自带的按钮完成。
- **单记录页**（新建注册页 / 详情页 / 审核注册页）：气泡保留直接操作。

## 四、注册填写智能辅助

> 覆盖填写期三类智能辅助：文件上传与信息填充、自动连通测试、自动生成产品与技术说明书。以下按角色（提供方 / Agent）说明：查看的页面 → 动作 / 步骤 → 操作按钮 → 交互说明 → 影响字段 / 元素。
> 

**数据权限**：Agent 仅在用户授权下读取本次上传 / 生成的材料，识别、填充与生成结果均作用于当前用户的当前表单，不跨记录、跨用户访问。

### 4.1 文件上传与信息填充

逻辑说明：用户在智能体对话窗口上传文件，Agent 识别文件内容并回填到注册表单对应字段；若上传的是 PDF 文件，同时自动归档到「备案材料」。

**1. 上传**：在 Agent 对话窗口点击【上传】或直接将文件拖拽 / “扔”进窗口任意区域上传（支持 PDF / Word / Excel / 图片 / 链接，单文件 ≤30M，超限或格式不符时提示），也可用文字 / 语音补充描述辅助识别。

**2. 识别**：Agent 对上传文件实时结构化识别（OCR / 解析），提取产品名称、版本、功能描述、接口地址、认证方式等关键信息；若识别到文件为 PDF，则自动归档到「备案材料」对应类别（产品说明书 / 技术规格书 / 其他材料）并标注来源。

**3. 回填并标注**：将识别结果回填至基本信息 / 技术信息对应字段，标注「AI 预填」与来源出处（敏感 / 存疑项标「请确认」供人工核对），并由已识别内容语义联动推断诊疗环节、所属科室等关联字段；随后用户在新建注册页逐项核对确认或修改，并在「备案材料」中预览、删除或补传已归档的 PDF。

### 4.2 自动连通测试

逻辑说明：技术信息中连通所需的必填项（接口地址 / 认证方式 / 密钥等）填写完成后，平台自动发起连通测试，无需手动点击；失败时给出诊断与解决步骤。

> 本功能仅限黑盒 API 接入方式。
> 

**1. 检查是否可自动测试**：仅黑盒 API 接入方式支持自动连通测试；平台检查新建注册页 · 技术信息区的连通必填项（接口地址 / 认证方式 / 密钥等）是否已完整，完整则进入自动测试，否则提示补全。

**2. 自动测试**：连通必填信息完整后由平台自动发起连通测试、无需手动点击，并以步骤条 / 时间线实时呈现 DNS 解析 → 建连 → 认证 → 请求 → 返回各阶段状态与耗时，在 Agent 对话窗口播报进度。

**3. 返回连通测试结果**：

3.1 连通成功弹「测试验证正常」；

3.2 连通失败：弹「测试验证异常」并给出错误代码与失败原因（网络不通 / 认证失败 / 参数不符 / 超时等）；Agent 随即联网搜索解决方案，在气泡中给出修改建议。用户修改后可重新测试，未通过前【提交】保持置灰。

### 4.3 自动生成产品与技术说明书

逻辑说明：当页面必填信息已完成、但未上传产品或技术文档时，Agent 主动提示用户是否需要生成产品 / 技术说明书；确认后依据当前已填信息生成文档，并在用户确认后自动归档到「备案材料」。

**1. 检查是否可生成材料**：Agent 检测页面必填信息是否已完成、且「备案材料」是否缺少产品 / 技术文档；当必填信息已完成但缺少对应文档时，进入生成流程。

**2. 气泡提示用户确定并生成材料**：

2.1 Agent 在对话窗口气泡主动提示是否需要生成对应文档，并说明将依据当前已填信息生成；

**Agent 气泡文案与交互：**

- **气泡文案（示例）**：「检测到当前『备案材料』还缺少产品说明书 / 技术说明书，我可依据你已填写的信息自动生成，需要现在生成吗？」
- **气泡按钮**：内置【生成产品说明书】【生成技术说明书】（可择一或同时点击）与【暂不生成】；已存在对应文档时，相应按钮不展示。
- **交互反馈**：点击生成后气泡进入「思考 / 生成中」态（动画 + 进度提示），生成完成后气泡提示：生成文档已上传到备案材料。

2.2 用户点击「生成产品说明书」或「生成技术说明书」（可择一或同时生成）确认后，Agent 依据已填基本信息 / 技术信息与已识别内容生成结构化的产品 / 技术说明书（PDF）供预览，确认无误后归档上传至「备案材料」（PDF、单文件 ≤30M）并在对话窗口反馈归档结果，如有需要可编辑或重新生成，也可在「备案材料」中预览、删除或补传。

**数据权限**：文档生成仅基于当前用户当前表单的已填信息与已上传材料，生成结果经用户确认后归档至本记录的「备案材料」，不跨记录、跨用户访问。

## 五、注册审核智能辅助

> 机制原则：智能助手承担预审与提示，管理员承担确认与决策，**最终决策权始终在管理员**。预审仅针对已填写的信息，不新增内容。
> 

### 5.1 审核内容智能预审

逻辑说明：信息科管理员进入审核注册页时，Agent 已提前完成注册信息预审并自动执行连通测试，管理员据此选择审核结果并填写审核结论；智能助手承担预审与提示，最终决策权始终在管理员。

**1. 智能预审注册信息**：

管理员打开审核注册页时，Agent 已完成逐项预审，对备案材料 / 基本信息 / 技术信息逐项检查；对信息预审采用就地标注——在对应字段旁以下划线标注疑似问题，并在旁边注明问题理由（必填缺失 / 不规范 / 前后不一致 / 材料与字段不匹配 / 时效等）；

（预审问题直接在页面对应字段旁就地标注，无需在气泡 / 对话窗口再次说明，也不在审核页新增独立面板 / 卡片 / 状态条 / 页面。预审仅针对已填信息，不新增、不修改内容。）

**2. 自动连通测试**：如果是API接入方式，Agent 对技术信息登记的接口自动执行连通测试，作为审核判断依据。

**3. 选择审核结果并填写结论**：**Agent** 根据问题标注与连通测试结果，给出「建议通过」或「建议退回」，并自动在审核结论区选中对应结果、把理由填入「具体说明」。后续管理员可直接采纳或修改。

**数据权限**：预审与审核仅针对管理员可见范围内、已提交待审的注册记录，审核阶段字段只读；Agent 仅读取已填信息用于预审标注与连通测试，不新增、不修改内容；审核（通过 / 退回并填写具体说明）为信息科管理员专属操作，普通用户不可审核；审核结论与具体说明写入该记录并同步至用户端「退回原因说明 / 具体说明」且仅提交人可见；审核操作自动归档留痕（审计中心）。
