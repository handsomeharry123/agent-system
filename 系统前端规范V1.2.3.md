# 系统前端规范 — 基于 Ant Design 5 统一组件规范要求

<aside>
📌

本规范基于 **Ant Design 5**，为系统所有模块（接入中心、评测沙盒、台账中心、编排协同、运行监控、安全治理、数据资产）提供统一的前端开发标准。所有 Claude 生成的 Demo 页面必须严格遵守，确保视觉风格、交互、代码结构一致。

</aside>

## 一、技术栈

- **React 18+**（函数组件 + Hooks，禁用 Class 组件）
- **Ant Design 5+**（唯一 UI 库，禁止混用）
- **@ant-design/pro-components 2+**（ProTable / ProForm / ProLayout）
- **@ant-design/icons 5+**（唯一图标库）
- **@ant-design/charts 2+**（图表）
- **TypeScript 5+**（strict 模式）
- **dayjs**（日期处理）

## 三、布局规范

### 3.1 整体结构（ProLayout）

- 侧边栏：展开 **240px** / 收起 **64px**，深色背景 `#001529`
- 顶栏高度：**64px**
- 内容区内边距：**24px**
- 卡片间距：同级 **16px**，区块 **24px**

### 3.2 栅格强制约束（最高优先级 🚨）

<aside>
🚨

**铁则：同一 `Row` 内所有 `Col` 的 `span` 之和必须 = 24。禁止留空、禁止固定宽度造成右侧空白。**

</aside>

| 同行子项数 | Col span | 典型场景 |
| --- | --- | --- |
| 1 | 24 | 单卡占整行 |
| 2 | 12 + 12 | 左右等分（CPU/GPU、双图对比） |
| 3 | 8 + 8 + 8 | 三等分（状态分布） |
| 4 | 6 × 4 | **四等分统计卡（最常见）** |
| 主辅 2:1 | 16 + 8 | 主图 + 侧边卡 |

**正确示例：**

```tsx
<Row gutter={[16, 16]}>
  <Col span={6}><Card style= height: '100%' ><Statistic title="平均时延" value="1820 ms" /></Card></Col>
  <Col span={6}><Card style= height: '100%' ><Statistic title="P95 时延" value="4450 ms" /></Card></Col>
  <Col span={6}><Card style= height: '100%' ><Statistic title="准确率" value="96.8%" /></Card></Col>
  <Col span={6}><Card style= height: '100%' ><Statistic title="可用率" value="99.95%" /></Card></Col>
</Row>
```

### 3.3 卡片与对齐

- 所有 `Card` 必须 `style= height: '100%'` ，确保同行等高
- Card 必须包裹在 `<Col span={N}>` 中，禁止裸用
- 统计卡片三段式（左对齐）：**标题 14px → 主数值 30px Semibold → 辅助文字 12px**，间距 `marginTop: 8`
- 趋势箭头 + 百分比同行，用 `<Space size={4}>` 包裹

### 3.4 Tab 内布局

- Tab 内第一个子节点必须是 `<Row gutter={[16, 16]}>`
- 单指标 Tab 必须配趋势图（`span={8} + span={16}`），或扩为 `span={24}`，**禁止单元素左对齐留白**

### 3.5 行间距统一

| 场景 | 间距 |
| --- | --- |
| 页面内边距 | 24px |
| 筛选栏 → 内容 | 16px |
| 同级 Row / 同行 Col | 16px（`gutter={[16, 16]}`） |

**禁止**：
``、空 div、`marginTop > 24px` 造成空隙；禁止不同 Row 用不同 gutter。

### 3.6 进度条列表（标签 + 进度 + 数值）

```tsx
<div style= display: 'flex', alignItems: 'center', gap: 12, marginBottom: 12 >
  <span style= flex: '0 0 120px' >agent-001</span>
  <Progress percent={65} style= flex: 1  showInfo={false} strokeColor="#52C41A" />
  <span style= flex: '0 0 60px', textAlign: 'right' >65%</span>
</div>
```

双组数据（CPU + GPU）必须 `<Col span={12}>` 左右对称。

## 六、核心组件规范

### 6.1 Button

- `type="primary"`：每个区域最多 1 个主操作
- `type="default"`：次要操作（取消/重置）
- `type="dashed"`：添加操作
- `type="text" / "link"`：轻量/导航操作
- `danger`：删除等不可逆操作
- 尺寸：页面级 `large`、默认 `middle`、表格内 `small`
- 按钮组：主按钮在右、取消在左
- 加载态用 `loading` 属性，图标按钮必配 `Tooltip`

### 6.2 表格（统一用 ProTable）

```tsx
<ProTable
  headerTitle="列表"
  rowKey="id"
  search= labelWidth: 'auto' 
  pagination={{
    defaultPageSize: 10,
    showSizeChanger: true,
    showQuickJumper: true,
    showTotal: (t) => `共 ${t} 条`,
  }}
  toolBarRender={() => [<Button type="primary" icon={<PlusOutlined />}>新建</Button>]}
  columns={columns}
  request={fetchData}
/>
```

- 操作列固定右侧（`fixed: 'right'`），用 `type="link"` 按钮 + `Space` 包裹
- 超长文字 `ellipsis: true` + `Tooltip`
- 空数据用 `Empty`

### 6.3 表单（统一用 ProForm）

- 默认水平布局，弹窗内垂直布局
- labelCol:wrapperCol = 6:18
- 失焦校验（`validateTrigger: 'onBlur'`）
- placeholder：输入「请输入xxx」、选择「请选择xxx」

### 6.4 Card 三铁律

1. **必须填满**：`style= height: '100%'` 
2. **防溢出**：`overflow: hidden`，长文本 `ellipsis` + `Tooltip`
3. **必须在 Row+Col 内**，禁止裸用

### 6.5 Modal 宽度

| 场景 | 宽度 |
| --- | --- |
| 确认弹窗 | 416px |
| 简单表单 | 520px |
| 复杂表单 | 720px |
| 详情/预览 | 960px |
| 复杂编辑 | 改用 Drawer |

### 6.6 反馈组件

- `message`：轻提示（3 秒）
- `notification`：重要通知
- `Modal.confirm`：二次确认
- `Alert`：静态提示
- `Result`：流程终态

## 七、导航

### 7.1 父级菜单冲突处理（强制）

<aside>
🚨

AntD `Menu` 中带 children 的 SubMenu **点击仅展开/收起，不导航**。父级若需访问，必须拆出「概览」子项，或父级配 `redirect` 到默认子路由。禁止 SubMenu 同时挂 children 和 link。

</aside>

### 7.2 侧边栏

- 层级最多 3 级
- 一级菜单带图标（`@ant-design/icons`），二/三级不带
- 菜单文字 ≤ 8 个中文
- 同时只展开一个子菜单

### 7.3 Tabs

- 页面级 `type="line"`
- 卡片内 `type="card"`
- 数量 2-7 个，超量用 `Dropdown`

## 十三、代码规范

### 13.1 命名

- 页面/组件：**PascalCase**（`AgentList.tsx`）
- 工具/Hooks：**camelCase**（`useAgent.ts`）
- 类型：`types.ts` / `*.d.ts`

### 13.2 目录结构

```
src/
├── components/    # 全局通用组件
├── layouts/       # 布局
├── pages/         # 按模块分目录
├── services/      # API
├── hooks/         # 自定义 Hooks
├── utils/         # 工具
├── theme/         # 主题 Token
└── types/         # 类型
```

### 13.3 样式书写优先级

1. AntD 组件属性（`size` / `type` / `bordered`）
2. Design Token（`theme.useToken()`）
3. CSS Modules

**禁止**：内联 `style`（动态值除外）、`!important`、覆盖 `.ant-*` 类名、全局 CSS
