# 系统前端规范 — 基于 Ant Design 5 统一组件规范要求

<aside>
📌

本规范基于 **Ant Design 5** 设计体系，为系统所有模块提供统一的前端开发标准。所有模块（智能体接入中心、评测沙盒、台账中心、编排协同、运行监控、安全治理、数据资产等）必须严格遵守本规范，确保 Claude 生成的 Demo 页面在视觉风格、交互行为、代码结构上完全一致。

</aside>

## 一、技术栈与依赖版本

| **依赖** | **版本要求** | **说明** |
| --- | --- | --- |
| React | ≥ 18.x | 函数组件 + Hooks，禁止 Class 组件 |
| Ant Design | ≥ 5.x | 唯一 UI 组件库，禁止混用其他 UI 库 |
| @ant-design/pro-components | ≥ 2.x | ProTable、ProForm、ProLayout 等复合组件 |
| @ant-design/icons | ≥ 5.x | 统一图标库，禁止使用第三方图标 |

| TypeScript | ≥ 5.x | 必须使用 TS，strict 模式开启 |
| dayjs | ≥ 1.x | 日期处理统一使用 dayjs（Ant Design 5 默认） |

## 二、全局主题配置（ConfigProvider）

<aside>
🎨

所有模块必须在根节点使用统一的 `ConfigProvider` 主题配置，**禁止在模块内部覆盖全局 Token**。

</aside>

### 2.1 Seed Token（种子令牌）

```tsx
import { ConfigProvider } from 'antd';

const themeConfig = {
  token: {
    // ===== 品牌色 =====
    colorPrimary: '#1677FF',        // 主色（蓝色系）
    colorSuccess: '#52C41A',        // 成功色
    colorWarning: '#FAAD14',        // 警告色
    colorError: '#FF4D4F',          // 错误色
    colorInfo: '#1677FF',           // 信息色

    // ===== 中性色基础 =====
    colorTextBase: '#000000',       // 文字基础色
    colorBgBase: '#FFFFFF',         // 背景基础色

    // ===== 字体 =====
    fontFamily: '-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif',
    fontSize: 14,                   // 基础字号

    // ===== 圆角 =====
    borderRadius: 6,                // 基础圆角

    // ===== 间距 =====
    sizeStep: 4,                    // 间距步长
    sizeUnit: 4,                    // 间距基础单位

    // ===== 线框模式 =====
    wireframe: false,               // 关闭线框模式
  },
};

// 应用入口
<ConfigProvider theme={themeConfig}>
  <App />
</ConfigProvider>
```

## 三、页面布局规范

### 3.1 整体布局结构（ProLayout）

```
┌─────────────────────────────────────────────────────┐
│  Header (64px)  │  Logo + 系统名称  │ 通知 ｜ 用户   │
├────────┬────────────────────────────────────────────┤
│        │  Breadcrumb 面包屑导航                      │
│ Side-  ├────────────────────────────────────────────┤
│  bar   │                                            │
│ (240px │         Content Area                       │
│  展开   │         内容区域                             │
│  64px  │         padding: 24px                      │
│  收起)  │                                            │
│        ├────────────────────────────────────────────┤
│        │  Footer (可选)                              │
└────────┴────────────────────────────────────────────┘
```

### 3.2 布局参数

| **参数** | **值** | **说明** |
| --- | --- | --- |
| 侧边栏宽度 (展开) | 240px | 使用 `Sider` 组件的 `width` 属性 |
| 侧边栏宽度 (收起) | 64px | 使用 `Sider` 组件的 `collapsedWidth` 属性 |
| 顶栏高度 | 64px | `Header` 固定高度 |
| 内容区内边距 | 24px | 上下左右统一 |
| 内容区最大宽度 | 不限制 | 自适应撑满（减去侧边栏） |
| 卡片间距 | 16px / 24px | 同级卡片间距 16px，区块间距 24px |
| 侧边栏背景色 | #001529 | 深色模式侧边栏（Ant Design 默认暗色） |

### 3.3 栅格强制约束（NON-NEGOTIABLE — 优先级最高）

<aside>
🚨

**核心问题诊断**：当前生成的页面频繁出现"卡片只占行宽 60%、右侧大片留白、Tab 内部内容塌缩、进度条只填左半"等布局事故。根本原因是没有强制遵循 24 列栅格"必须占满"原则。本节为**最高优先级强制约束**，所有页面必须严格遵守，否则视为不合格代码必须重写。

</aside>

**铁则：同一 `Row` 内所有 `Col` 的 `span` 之和必须恰好等于 24，禁止留空、禁止「自适应宽度」导致右侧空白。**

| **同行子项数** | **Col span** | **典型场景** |
| --- | --- | --- |
| 1 | span={24} | 单卡占整行（主图表、表格） |
| 2 | span={12} | 左右等分（CPU/GPU、双图对比） |
| 3 | span={8} | 三等分（指标卡、状态分布） |
| 4 | span={6} | **四等分统计卡片（最常见）** |
| 2:1 主辅 | span={16} + span={8} | 主图表+侧边小图/统计 |
| 6 | span={4} | 六等分（多状态卡） |

**❌ 反面案例（严禁出现，对应当前截图问题）：**

- ❌ 4 个统计卡片只占页面左侧 ~70%，右侧大片空白
- ❌ "回答准确率 96.8%" 独占 Tab 左半，右侧空着
- ❌ "GPU 显存使用率" 列表只填左半，右半空白
- ❌ "不合规答率 0.32%" 单独成块，旁边巨大空白
- ❌ 使用固定 `width: 240px` 让卡片"自适应"导致非整数倍空隙

**✅ 正确做法（代码模板）：**

```tsx
// ✅ 4 个统计卡片：必须 Col span={6}，合计 24，无空隙
<Row gutter={[16, 16]}>
  <Col span={6}><Card style= height: '100%' ><Statistic title="平均响应时延" value="1820 ms" /></Card></Col>
  <Col span={6}><Card style= height: '100%' ><Statistic title="P95 响应时延" value="4450 ms" /></Card></Col>
  <Col span={6}><Card style= height: '100%' ><Statistic title="回答准确率" value="96.8%" /></Card></Col>
  <Col span={6}><Card style= height: '100%' ><Statistic title="服务可用率" value="99.95%" /></Card></Col>
</Row>

// ✅ CPU/GPU 资源列表：必须 Col span={12} 左右对称
<Row gutter={[16, 16]}>
  <Col span={12}><Card title="CPU 使用率">{/* 进度条列表 */}</Card></Col>
  <Col span={12}><Card title="GPU 显存使用率">{/* 进度条列表 */}</Card></Col>
</Row>

// ✅ Tab 内单指标：必须配辅助图或扩为整行
<Row gutter={[16, 16]}>
  <Col span={8}><Card><Statistic title="回答准确率" value="96.8%" /></Card></Col>
  <Col span={16}><Card title="准确率趋势"><Line {...config} /></Card></Col>
</Row>
```

### 3.4 同行卡片等高约束

- 同一 `Row` 内所有 `Card` 必须等高。`Row` 默认 `align="stretch"`，不要改为 `align="top"`。
- 每个 `Card` 必须设置 `style= height: '100%'` ，确保撑满 `Col` 容器。
- 卡片内"主数值 + 辅助文字"的总占位高度需保持一致；内容不足时使用 `<div style= minHeight: 80 >` 占位，禁止让卡片"缩水"。

### 3.5 卡片内部三段式垂直对齐

所有统计卡片必须采用**三段式垂直对齐**，禁止居中、禁止漂浮：

```
┌────────────────────────────────┐
│ ① 标题   colorTextSecondary, 14px, 单行省略 │
│ ② 主数值 30px Semibold, 主题色或状态色      │
│ ③ 辅助说明 12px, colorTextTertiary       │
│     （趋势箭头与百分比同行，禁止换行）       │
└────────────────────────────────┘
```

- 三段之间使用 `marginTop: 8px`，禁用空行/
``占位。
- 主数值与标题、辅助说明全部**左对齐**，禁止居中。
- 趋势箭头（↑↓）必须与百分比同一行；图标 + 文字使用 `<Space size={4}>` 包裹。

### 3.6 Tab 内部布局强约束

<aside>
⚠️

**Tab 切换后，内部内容必须使用与外部相同的 24 列栅格系统对齐，不允许塌缩为左对齐窄列。**

</aside>

- Tab 内的第一个子节点必须是 `<Row gutter={[16, 16]}>`。
- Tab 内的标准布局选项（必须选其一，禁止单元素左对齐留白）：
    - 单图全宽：`<Col span={24}>`
    - 主图 + 侧边卡：`<Col span={16}>` + `<Col span={8}>`
    - 双图等分：`<Col span={12}>` × 2
    - 三图等分：`<Col span={8}>` × 3
    - 指标卡 + 主图：`<Col span={8}>` + `<Col span={16}>`
- **禁止**：Tab 内只显示一个"准确率 96.8%"卡却让右侧空白 → 必须配趋势图或扩 `span={24}` 居中数值。

### 3.7 行间距与区块间距统一

| **场景** | **间距** | **实现** |
| --- | --- | --- |
| 页面整体内边距 | 24px | ProLayout `contentStyle: padding: 24`  |
| 顶部筛选栏 → 内容 | 16px | 筛选栏 Card 的 `marginBottom: 16` |
| 同级 Row 之间 | 16px | 外层 Row 的 `gutter={[16, 16]}` 或 `marginBottom: 16` |
| 同行 Col 之间 | 16px | Row 的 `gutter={[16, 16]}` |
| Tab 顶部 → 内容区 | 16px | Tabs 内容容器 `marginTop: 16` |

**禁止事项：**

- ❌ 在 Row 之间插入 
``、空 `<div>` 或 `<Divider style= margin: '32px 0' />`
- ❌ 使用 `marginTop > 24px` 的"大空隙"分隔卡片与图表
- ❌ 卡片底部内边距超过 24px
- ❌ 不同 Row 使用不同的 gutter（如 [16,16] 与 [24,24] 混用）

### 3.8 进度条/列表类组件占位规则

针对 CPU 使用率、GPU 显存、资源占用排行等"标签 + 进度条 + 数值"型列表：

- **列表容器必须填满 Card body 宽度**。
- 若同时存在两组数据（CPU + GPU），必须使用 `<Row gutter={[16,16]}>` + 双 `<Col span={12}>` 左右对称。
- 若仅有一组数据但 Card 较宽（>600px），使用两列网格：每行 2 条进度条，每条 `<Col span={12}>`。
- 单条进度条 Flex 布局（强制）：

```tsx
<div style= display: 'flex', alignItems: 'center', gap: 12, marginBottom: 12 >
  <span style= flex: '0 0 120px', color: token.colorTextSecondary >agent-001-实例1</span>
  <Progress percent={65} style= flex: 1 ` showInfo={false} strokeColor="#52C41A" />
  <span style= flex: '0 0 60px', textAlign: 'right' >65%</span>
</div>
```

## 六、核心组件使用规范

### 6.1 按钮 Button

| **变体** | **Antd 属性** | **使用场景** |
| --- | --- | --- |
| 主要按钮 | `type="primary"` | 页面主操作（提交、确认、新建），每个区域最多 1 个 |
| 默认按钮 | `type="default"` | 次要操作（取消、重置、返回） |
| 虚线按钮 | `type="dashed"` | 添加操作（新增行、新增项） |
| 文本按钮 | `type="text"` | 轻量操作（编辑、查看详情） |
| 链接按钮 | `type="link"` | 导航型操作（跳转、外链） |
| 危险按钮 | `danger` | 删除、停用等不可逆操作 |

**尺寸规范：**

- 大 `size="large"` (40px) → 页面级主操作
- 中 `size="middle"` (32px) → 默认，表单/卡片内操作
- 小 `size="small"` (24px) → 表格内操作、紧凑区域

**强制规则：**

- 按钮组中主要按钮在右侧，取消按钮在左侧
- 加载态必须使用 `loading` 属性，禁止自定义 Spinner
- 图标按钮必须搭配 `Tooltip` 提供文字说明

### 6.2 表格 Table / ProTable

<aside>
📊

所有列表页统一使用 `ProTable` 组件，禁止使用原生 `Table` 手动拼装搜索/分页。

</aside>

**标准结构：**

```tsx
<ProTable
  headerTitle="智能体列表"              // 表格标题
  rowKey="id"
  search= labelWidth: 'auto'       // 搜索表单
  pagination={{
    defaultPageSize: 10,               // 默认每页 10 条
    showSizeChanger: true,             // 允许切换每页条数
    showQuickJumper: true,             // 允许快速跳页
    pageSizeOptions: ['10', '20', '50', '100'],
    showTotal: (total) => `共 ${total} 条`,
  }}
  toolBarRender={() => [
    <Button type="primary" icon={<PlusOutlined />}>
      新建
    </Button>,
  ]}
  columns={columns}
  request={fetchData}
/>
```

**表格规范：**

- 行高：默认 54px（Ant Design 标准）
- 表头背景：`#FAFAFA`（`colorBgContainerDisabled`）
- 表头文字：`colorTextSecondary`，`fontWeight: 600`
- 操作列固定右侧：`fixed: 'right'`
- 操作按钮用 `Space` 包裹，统一使用 `type="link"` 按钮
- 空数据使用 `Empty` 组件，配文字说明
- 超长文字使用 `Tooltip` + `ellipsis: true`

### 6.3 表单 Form / ProForm

**统一使用规则：**

```tsx
<ProForm
  layout="horizontal"                  // 水平布局为默认
  labelCol= span: 6                // label 宽度
  wrapperCol= span: 18             // 控件宽度
  requiredMark="optional"              // 显示「可选」而非必填星号
  submitter={{
    searchConfig: {
      submitText: '提交',
      resetText: '重置',
    },
  }}
>
```

**表单规范：**

| **规则** | **要求** |
| --- | --- |
| 布局 | 默认水平布局 `horizontal`，弹窗内使用垂直布局 `vertical` |
| 标签宽度 | 统一 6:18 比例，或使用 `labelWidth: 'auto'` |
| 必填提示 | 使用 `rules` 的 `required`  • `message` 属性 |
| 校验时机 | `validateTrigger: 'onBlur'`（失焦校验，非实时） |
| 错误提示 | 控件下方红色文字（Ant Design 默认行为，禁止自定义位置） |
| placeholder | Input 用「请输入xxx」，Select 用「请选择xxx」 |

### 6.4 卡片 Card

<aside>
⚠️

**Card 强制规则（三条铁律）**：

1. **必须填满**：Card 必须设置 `style= height: '100%'` ，确保撑满所在 Col。禁止 Card 出现"未填满 Col 导致右侧留白"。
2. **防溢出**：包含动态数据的 Card 设置 `overflow: hidden`，长文本使用 `ellipsis` + `Tooltip`。禁止内容撑破卡片。
3. **必须在 Row+Col 内**：Card **不允许**作为顶层裸节点出现，必须包裹在 `<Col span={N}>` 中（N 由同级子项数推算，见 §3.3）。
</aside>

```tsx
<Card
  title="模块标题"                    // 标题
  extra={<Button type="link">更多</Button>}  // 右上角操作
  bordered={false}                    // 无边框风格
  style= marginBottom: 16         // 卡片间距 16px
>
  {/* 内容 */}
</Card>
```

**卡片规范：**

- 内边距：`24px`（Ant Design Card 默认）
- 背景色：`#FFFFFF`（colorBgContainer）
- 圆角：`8px`（borderRadiusLG）
- 阴影：不使用阴影，使用 `bordered={false}` + 页面灰色底
- 标题字号：16px，`fontWeight: 500`

### 6.5 弹窗 Modal

| **场景** | **宽度** | **说明** |
| --- | --- | --- |
| 确认弹窗 | 416px | 使用 `Modal.confirm()`，简短确认信息 |
| 表单弹窗 (简单) | 520px | 3-5 个字段的表单 |
| 表单弹窗 (复杂) | 720px | 多步骤表单、6+ 个字段 |

## 七、导航规范

### 7.0 父级菜单与页面导航冲突处理（强制规则）

<aside>
🚨

Ant Design 的 `Menu` / `ProLayout` 中，**带有 children 的菜单项（SubMenu）点击时仅展开/收起子菜单，不会触发页面导航**。因此，若某个功能既需要作为可访问页面、又包含子页面，必须采用以下方案之一：

</aside>

**方案 A（推荐）：拆分为子菜单项**

将父级页面内容独立为一个子菜单项（如"概览"），父级仅作为分组容器：

```
├── 首页（SubMenu，不可直接导航）
│   ├── 首页概览（MenuItem，默认落地页） ← 原首页内容
│   └── 数据大屏（MenuItem）
```

路由配置：父级路由（如 `/app/home`）设置 `redirect` 到默认子路由（如 `/app/home/overview`）。

**方案 B：独立菜单 + 页内跳转**

父级作为独立的一级菜单（无 children），子页面通过页面内部链接/按钮进入，不在侧边栏展示。适用于子页面使用频率低的场景。

**禁止**：将同一个菜单项同时配置为 SubMenu（有 children）和 Link（可导航），此写法在 Ant Design 中不生效。

### 7.1 侧边栏菜单

```tsx
// 菜单层级最多 3 级
const menuItems = [
  {
    key: 'agent-center',
    icon: <RobotOutlined />,
    label: '智能体接入中心',
    children: [
      { key: 'register', label: '注册备案' },
      { key: 'auth', label: '身份认证' },
      { key: 'permission', label: '权限管理' },
    ],
  },
  // ...
];
```

**强制规则：**

- 一级菜单必须带图标（使用 `@ant-design/icons`）
- 二级/三级菜单不带图标
- 菜单文字最长 8 个中文字符
- 当前页菜单项高亮：使用 `selectedKeys` 绑定路由
- 展开菜单使用 `openKeys` 控制，同时只展开一个子菜单

### 7.2 面包屑

```tsx
<Breadcrumb items={[
  { title: '首页', href: '/' },
  { title: '智能体接入中心' },
  { title: '注册备案' },        // 最后一级不可点击
]} />
```

### 7.3 标签页 Tabs

- 页面级 Tab：使用 `type="line"`（线性样式）
- 卡片内 Tab：使用 `type="card"`（卡片样式）
- Tab 数量建议 2-7 个，超过使用 `Dropdown` 收起
