# 系统配置-需求说明

面向平台管理员提供**系统级基础配置能力**，统一维护平台运行所依赖的**数据字典**与**大模型**资源。系统配置模块包含两个二级模块：**数据字典**（维护科室、诊疗环节、资源列表等基础字典及字典项）与**模型配置**（登记、测试并接入平台可调用的大模型），为需求管理、接入注册、台账、监控等业务模块提供统一的字典取值与模型能力支撑。

<aside>
ℹ️

**当前阶段说明**：本文档字段与交互均依据《系统配置具体字段及呈现形式》整理；字典、字典项、模型信息均由管理员在对应页面**手动填写、选择或通过文件导入**完成。

</aside>

<aside>
🧭

**菜单结构与导航**：「系统配置」为一级菜单，下设 **数据字典** 与 **模型配置** 两个二级模块。

- **数据字典**：默认进入**字典管理页**，通过行内【查看字典项】进入**字典项管理页**；【新增字典】【新增字典项】分别下拉进入「创建 / 导入」类子页面。
- **模型配置**：默认进入**模型管理页**，通过右上角【模型配置】进入**模型配置页**。
- 创建 / 导入 / 配置类页面均为页面级子路由，通过按钮进入，不在菜单中单独挂出。
</aside>

<aside>
🔒

**访问范围**：系统配置面向平台**信息科 IT 管理员**开放，用于维护全院统一的字典与模型资源；配置结果全局生效，供各业务模块引用。

</aside>
