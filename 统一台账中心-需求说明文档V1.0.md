# 统一台账中心-需求说明文档

### 模块定位

集中展示智能体核心信息（名称、类型、科室、供应商、版本、模型、算力部署等），支持筛选与搜索，实现全院智能体全盘掌握。

### 功能说明

| **一级功能** | **二级功能** | **功能说明** |
| --- | --- | --- |
| 智能体信息展示 | 核心信息集中可视化展示 | 以列表/卡片形式展示智能体名称、类型、归属科室、供应商、版本、模型来源、算力部署方式、运行状态、使用日志等 |
| 台账检索 | 智能体筛选与搜索 | 支持按科室、类型、供应商、状态等多条件筛选，以及关键字快速搜索 |

### 核心页面清单

| **页面名称** | **页面类型** | **主要用途** | **使用角色** |
| --- | --- | --- | --- |
| 台账总览页 | 统计+导航页 | 智能体总数与状态分布统计、科室分布、快捷入口 | 平台管理员
科室管理员（本科室） |
| 台账列表页 | 列表页 | 多条件筛选与搜索，查看全院智能体核心信息 | 平台管理员（全部）
科室管理员（本科室） |
| 台账详情页 | 详情页 | 单个智能体完整信息、状态变更时间线、关联跳转 | 平台管理员（可编辑）
科室管理员（只读） |

### 台账状态设计

台账区分**生命周期状态**和**运行状态**两个维度：

| **维度** | **状态值** | **说明** |
| --- | --- | --- |
| 生命周期状态 | 已接入 / 评测中 / 试运行中 / 已上线 / 已注销 | 由各模块自动同步 |
| 运行状态 | 在线 / 离线 / 异常 | 仅「已上线」时显示，由监控中心实时同步 |

### 4-1 台账总览页 — 字段与交互

#### 页面概述

| 属性 | 说明 |
| --- | --- |
| 页面类型 | 统计 + 导航页 |
| 使用角色 | 平台管理员（全院数据）、科室管理员（本科室数据） |
| 对应功能 | 智能体核心信息集中可视化展示 |
| 入口 | 平台首页侧边栏「统一台账中心」 |

#### 设计原则

- **首屏完整展示**：核心统计与全部分布图表在 1440×800 视口内首屏可见，无需滚动。
- **信息密度优先**：紧凑横向卡片 + 单行多图表布局，最大化单屏信息呈现。
- **视觉层次清晰**：自上而下「标题操作栏 → 数据卡片 → 分布图表」三段式，节奏统一。
- **去冗余**：取消独立的「快捷操作」分区，操作按钮上提到页头右侧；图表卡片去多余留白、统一高度。

#### 页面布局（参考视口 1440×800，内容区高度 ≤ 720px）

│ 统一台账中心  查看所有智能体台账信息       [查看列表] [导出台账] │ 56px

│ ┌─总数─┐ ┌─已上线─┐ ┌─评测/试运行─┐ ┌─异常告警─┐ ┌─已注销─┐ │ 88px

│ │   环形图    │ │  横向条形  │ │  环形图  │ │  折线图   │ │

- 右侧：操作按钮组「查看台账列表」（主按钮）、「导出台账」（次按钮，仅平台管理员可见）。

**区域 1：统计卡片**（高度 88px，单行 5 等分）

紧凑横向卡片，每张卡片左侧图标 + 右侧数值与标签，去除装饰性背景。

- 左侧 32×32 圆形浅底图标，右侧上行数值（28px 加粗），下行标签（13px 次要色）。
- 鼠标 hover 时整卡有浅阴影 + 光标小手，强调可点击。

**区域 2：分布图表**（高度 340px，单行 4 等分）

所有图表卡片宽度均等（约 24% 宽度），统一高度 340px，标题置于卡片左上角（14px），图例右上角小标签样式。

| **序号** | **图表名称** | **图表类型** | **说明与设计要点** |
| --- | --- | --- | --- |
| 1 | 生命周期状态分布 | 环形图（Donut） | 各生命周期状态占比；圆环中心显示「总数 30」加粗大字，下方小字「智能体」；图例 5 项两列排布于卡片底部 |
| 2 | 科室分布 Top10 | 横向条形图 | 取数量最多的前 10 个科室，纵轴为科室名（中文较长，横向条形便于阅读），按数量倒序；条形末端标注数值 |
| 3 | 类型分布 | 环形图（Donut） | 按智能体类型占比展示，类型数量较多时合并 Top5 + 其他；图例置于卡片底部两列 |
| 4 | 接入趋势 | 折线图 | 近 6 个月新增接入数量；X 轴月份，Y 轴数量；折线 + 渐变面积填充，节点显示数值 |

图表通用设计要点：

- 卡片标题字号 14px、副标题/图例 12px，留白控制在 12–16px。
- 主色板与全局保持一致：蓝 / 绿 / 橙 / 紫 / 红 / 灰，避免随机配色。
- 图表内部去除冗余坐标网格线，仅保留必要刻度。
- 在 ≤1280px 视口下，区域 2 自动折行为「2 行 ×2 列」作为降级方案。

### 4-2 台账列表页 — 字段与交互

#### 页面概述

| 属性 | 说明 |
| --- | --- |
| 页面类型 | 列表页 |
| 使用角色 | 平台管理员（全部）、科室管理员（本科室） |
| 对应功能 | 智能体筛选与搜索 |
| 入口 | 台账总览页「查看台账列表」/ 侧边栏 |

#### 设计原则

- **筛选区单行优先**：常用筛选项默认单行展示，溢出项点击「展开 ⌄」收起；操作按钮始终固定在第一行右侧。
- **组件宽度统一**：下拉筛选统一 160px，搜索框 240px，日期范围 280px，所有筛选组件高度统一 32px。
- **列表横向不溢出**：名称列左侧固定、操作列右侧固定，中间区域横向滚动；两个固定列带 1px 阴影分隔。
- **长文本截断**：供应商、模型来源等列固定列宽，溢出 `...` + Tooltip 完整展示。
- **操作列防溢出**：主操作直显，次要操作收入「···」更多菜单；操作列宽度固定，绝不超过列宽。

#### 筛选区布局

```jsx
┌─────────────────────────────────────────────────────────────────┐
│ [关键字搜索 240] [生命周期 160] [运行状态 160] [科室 160] [类型 160]│ 行1
│ [展开 ⌄]                       [重置] [列表|卡片] [导出 Excel]   │
├─────────────────────────────────────────────────────────────────┤
│ [供应商 160] [部署形态 160] [模型来源 160] [接入时间 280]         │ 行2(展开)
└─────────────────────────────────────────────────────────────────┘
```

- **第一行（始终显示）**：左侧 = 关键字搜索 + 4 个最常用筛选 + 「展开 ⌄」切换；右侧固定 = 「重置」「列表/卡片切换」「导出 Excel」（仅平台管理员）。
- **第二行（展开后显示）**：供应商、部署形态、模型来源、接入时间区间；点击「收起 ⌃」隐藏第二行。
- 筛选项之间间距 8px，行间距 12px；当某筛选项有值时，「展开」按钮旁显示已选数量徽标（如 `展开 ⌄ (2)`）。

| 2 | 生命周期状态 | 1 | 160px | 下拉筛选 | 全部 / 已接入 / 评测中 / 试运行中 / 已上线 / 已注销 |
| 3 | 运行状态 | 1 | 160px | 下拉筛选 | 全部 / 在线 / 离线 / 异常 |

#### 列表布局规范

- 表头高度 44px，行高 52px，行间分隔线 1px 浅灰。
- **左固定列**：智能体名称；**右固定列**：操作；两固定列右/左外侧各加 1px 阴影分隔，与滚动区视觉区分。
- 中间字段区域横向可滚动；当列总宽 ≤ 视口可用宽度时不显示滚动条。
- 长文本列固定列宽，超出 `...` + hover Tooltip 显示完整内容。
- 标签类列（类型、模型来源、部署形态、状态）统一使用 Tag 组件，圆角 4px，高度 22px。

#### 操作列规范

- **主操作直显**：「详情」（所有角色可见）。
- **次要操作按角色/状态显示**：
    - 「注销」：生命周期为「已上线 / 试运行中」时显示，可见角色为本科室科室管理员、平台管理员。
    - 「监控」：生命周期为「已上线」时显示，仅平台管理员可见。
- **「···」更多菜单**：当行内可见操作超过 2 个时，第 3 个起统一收入「···」下拉菜单（如 复制信息、查看评测、版本历史 等）。
- 操作之间使用 8px 间距 + 竖向分隔符；图标 + 文字组合，单按钮宽度 ≤ 48px。
- 操作列固定宽度 180px，背景与表格一致；右侧 1px 阴影分隔滚动区。

### Mock 数据样例（全局对象）

本节定义统一台账中心模块的**全局 Mock 数据对象**，作为前端 Demo、原型联调与后端接口契约的统一数据源。总览页、列表页、详情页均从此对象消费数据；字段命名与上文 4-x 各节一致，枚举与项目文档、整体需求文档保持同步。

#### 设计约定

- **唯一数据源**：总览页统计数字、各分布图表、列表行、详情页字段均派生自 `agents` 数组，统计可在前端动态计算。
- **枚举集中维护**：下拉选项、状态颜色映射统一放在 `ENUMS` 与 `STATUS_COLOR`，不在页面硬编码。
- **权限上下文**：`currentUser.role` 决定可见范围（平台管理员见全院，科室管理员仅见本科室），在前端根据 `currentUser.department` 过滤列表与统计。
- **状态三维度**：`lifecycleStatus`（生命周期）、`runtimeStatus`（仅「已上线」有效）、`accessStatus`（来自接入中心，用于贯通接入阶段）三维度独立，不要合并。

#### 文件位置建议

```jsx
src/mock/ledger.ts        // 本节定义的全局对象
src/pages/ledger/         // 总览页 / 列表页 / 详情页从 mock 消费数据
```

#### 使用示例

```tsx
import { agents, currentUser, getStatistics, getVisibleAgents, STATUS_COLOR } from '@/mock/ledger';

// ① 总览页 - 统计卡片
const stats = getStatistics();
// => { total: 3, online: 1, evaluating: 1, abnormal: 0, deregistered: 1 }

// ② 列表页 - 按角色过滤 + 状态标签颜色
const rows = getVisibleAgents();
rows.map(a => ({ ...a, lifecycleTagColor: STATUS_COLOR.lifecycle[a.lifecycleStatus] }));

// ③ 详情页 - 按 id 查询
const agent = agents.find(a => a.id === 'AGT-2024-001');
```

#### 与其他模块的字段来源

| **字段 / 对象** | **主数据源模块** | **同步时机** |
| --- | --- | --- |
| 智能体基本信息、技术信息、接入时间 | 智能体接入中心（模块 4） | 接入成功后同步创建、信息变更后增量同步 |
| `lifecycleStatus`、`evaluationReportId` | 统一准入评测沙盒（模块 6） | 评测/试运行/上线节点变更时推送 |
| `runtimeStatus` | 统一运行监控中心（模块 8） | 实时同步，仅 lifecycleStatus = 「已上线」时有效 |
| `relatedFlows` | 多智能体编排协同中心（模块 7） | 编排流程创建/修改时增量同步 |
| `lifecycleTimeline` | 各模块状态事件汇聚 | 按事件发生顺序追加，不可修改历史记录 |
| `currentUser` | 用户中心（模块 11） | 登录后从 Session/Token 解析注入 |
